# P04 — HTTP API (`api/`) spec

**Status:** in-flight (App-Build Step 1 — spec for sign-off).
**Companion docs:** [Design §6.5](../superpowers/specs/2026-04-27-mame-curator-design.md) · [Long-form roadmap Phase 4](../superpowers/specs/2026-04-27-roadmap.md) · [Coding standards](../standards/coding-standards.md).

## Contract

Given a fully-loaded backend (parsed DAT + 5 INIs + `cloneof_map` + `bios_chain` + initial `FilterResult`), this module exposes that state and the P03 copy pipeline over HTTP for the Phase 6 frontend (and any other HTTP client). Specifically:

1. **Read** the curated library — games, alternatives, explanations, stats — from the cached `FilterResult`.
2. **Edit** overrides, sessions, per-game notes, and config — every write atomically snapshots the prior file to `data/snapshots/<ts>/` and recomputes the cached `FilterResult` synchronously.
3. **Run copy jobs** via P03's `run_copy`, streaming live progress over Server-Sent Events; pause / resume / cancel routed through the existing `CopyController`.
4. **Sandbox filesystem browsing** for the wizard's Browse buttons and the Settings paths editor — strict allowlist by default, runtime-grantable through a dedicated API.
5. **Stub** the routes whose owning module ships in P07 (`updates`) or P08 (`setup`) — stable response shapes, hardcoded payloads, no future-phase logic.

The CLI surface is `mame-curator serve [--host 127.0.0.1] [--port 8080] [--config <path>]`; defaults match `config.example.yaml`. The module depends on `parser/`, `filter/`, `copy/`, `cli/`. It MUST NOT import from `media/`, `updates/`, `help/`, or `setup/` per the project's anti-jump rule.

## Architecture

```
                ┌─────────────────────────────────────────┐
   browser ───► │ FastAPI app (uvicorn, 127.0.0.1:8080)   │
                │  ┌──────────┐  ┌──────────┐  ┌────────┐ │
                │  │ routes.py│  │ jobs.py  │  │ fs.py  │ │
                │  │ (HTTP)   │  │ (copy)   │  │ (sand) │ │
                │  └────┬─────┘  └────┬─────┘  └───┬────┘ │
                │       │             │            │      │
                │       ▼             ▼            ▼      │
                │     app.state.world (frozen blob)       │
                │     app.state.job   (singleton, mutable)│
                └─────────────────────────────────────────┘
                            │
                            ▼
                  parser/ filter/ copy/ (synchronous library code)
```

**Single mutable `app.state.job`** holds the in-flight copy job (or `None`); singletons by design — the second concurrent `POST /api/copy/start` returns `409 Conflict`. **Frozen `app.state.world`** holds parsed-data + `FilterResult`; PATCH `/api/config` swaps the whole blob atomically (read paths see consistent state). Lifespan startup pays the parse + filter cost (~1–2 s on the real 43k-machine DAT); reads are zero-compute.

## Lifespan + `app.state` model

```python
class WorldState(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid", arbitrary_types_allowed=True)

    config: AppConfig
    machines: Mapping[str, Machine]            # MappingProxyType wrapping the parser dict
    cloneof_map: Mapping[str, str]
    bios_chain: Mapping[str, BIOSChainEntry]
    ctx: FilterContext                         # FP05 B6 mappings — owner-mutable, treated read-only
    overrides: Overrides
    sessions: Sessions
    filter_result: FilterResult
    notes: Mapping[str, str]                   # data/notes.json contents
    allowed_roots: tuple[FsAllowedRoot, ...]   # composed allowlist; cached on world (§ Sandbox)
```

**Atomicity caveat.** `app.state.world` is swapped via single attribute assignment; under CPython's GIL this is atomic (no torn read possible). The project targets CPython 3.12+ only. A future no-GIL CPython (PEP 703 free-threaded build) would require either a `threading.Lock` on read or a `contextvars`-style swap; revisit at that time.

`lifespan` startup:

1. Load `config.yaml` from `--config` (or `./config.yaml`); parse via `AppConfig` Pydantic model. Raise `ConfigError` if missing or invalid; uvicorn bails before binding the port.
2. Run `parse_dat`, `parse_catver`, `parse_languages`, `parse_bestgames`, `parse_mature`, `parse_series`, `parse_listxml_cloneof`, `parse_listxml_disks`, `parse_listxml_bios_chain` against the configured paths. Wrap any `ParserError` into `ApiStartupError` with the failing path quoted via `repr()` per FP07 A5.
3. Load `overrides.yaml` and `sessions.yaml` via the existing loaders.
4. Load `data/notes.json` (default: `{}` if missing).
5. Build `FilterContext` and run `run_filter` → `FilterResult`.
6. Construct `WorldState`; assign to `app.state.world`.
7. Initialize `app.state.job: JobManager`. (`JobManager` itself is created here but holds no job yet.)
8. Yield.

Shutdown:

1. If a job is running, signal cancel via `CopyController.cancel()` and wait up to 5 s for the worker thread to finish; force-detach if it doesn't.
2. Close the SSE event queue.

### Re-computation triggers

Any of the following swap `app.state.world` under a single `asyncio.Lock` guarding writes (reads are unlocked — `WorldState` is frozen, swap is atomic via Python attribute reassignment):

| Trigger | What re-runs |
|---|---|
| `PATCH /api/config` | Re-validate + atomic-write `config.yaml` → snapshot prior → re-build `FilterContext` (if filter-relevant fields changed) → re-run `run_filter` → swap `world`. |
| `POST /api/overrides`, `DELETE /api/overrides/{parent}` | Atomic-write `overrides.yaml` → snapshot prior → re-run `run_filter` → swap `world`. |
| `POST/DELETE /api/sessions[/{name}]`, `POST /api/sessions/{name}/activate` | Atomic-write `sessions.yaml` → snapshot prior → re-run `run_filter` → swap `world`. |
| `PUT /api/games/{name}/notes` | Atomic-write `data/notes.json` → snapshot prior → swap `world` (filter unchanged). |
| `POST /api/config/snapshots/{id}/restore` | Read snapshot → atomic-write current files → re-build → swap `world`. |
| `POST /api/fs/allowed-roots`, `DELETE /api/fs/allowed-roots/{id}` | Atomic-write `config.yaml` (`fs.granted_roots`) → snapshot → swap `world` (filter unchanged). |

The write-lock is **per-app**, not per-resource; concurrent PATCHes serialize. Acceptable: writes are rare and short.

## JobManager + SSE contract

### Class surface

```python
class JobManager:
    """Singleton manager for the in-flight copy job."""

    def __init__(self) -> None: ...

    @property
    def current(self) -> Job | None: ...

    async def start(self, plan: CopyPlan) -> Job:
        """Spawn the worker thread; raise JobAlreadyRunningError if one exists."""

    async def pause(self) -> JobStatus: ...
    async def resume(self) -> JobStatus: ...
    async def abort(self, *, recycle_partial: bool) -> JobStatus: ...

    def events(self) -> AsyncIterator[JobEvent]:
        """Return a fresh subscriber async-iterator. Each call gets its own
        bounded queue (multi-subscriber fan-out — see below). Iteration ends
        when the job terminates (job_finished or job_aborted is the last event
        the consumer sees, then StopAsyncIteration). Implementation is a
        plain async-generator hidden behind this sync entry point so the
        return type is a true AsyncIterator, not an awaitable returning one."""


@dataclass(frozen=True, slots=True)
class Job:
    id: str                                # ULID-ish: strftime + token_hex(4), matching copy/runner.py:45
    plan: CopyPlan
    started_at: datetime
    controller: CopyController             # P03's existing controller — pause/cancel pass through
    thread: threading.Thread               # the run_copy worker
    queue: asyncio.Queue[JobEvent]         # bounded; full-queue policy = drop-oldest
```

### SSE event types

`run_copy` (`src/mame_curator/copy/runner.py:62-66`) only exposes one hook: `on_progress: Callable[[str, int, int], None]`. Every event below has a documented source — either JobManager owns its emission directly, or it is **synthesized** from `on_progress` transitions / controller state at progress ticks. **No new callbacks are added to `run_copy`** (anti-jump rule 2 + § Out of scope last bullet).

Discriminated union via `event: Literal[...]`:

| Event | Payload | Source |
|---|---|---|
| `job_started` | `{job_id, total_files, total_bytes, started_at}` | JobManager, before spawning the worker thread. |
| `bios_warning` | `{name, kind}` (forwarded from `BIOSResolutionWarning`) | JobManager, **before** spawning the worker thread. JobManager calls `resolve_bios_dependencies(plan.winners, world.bios_chain)` itself first (already a public function in `copy/__init__.py`); replays each warning as an SSE event; then proceeds. This means BIOS warnings reach the UI before any file copy starts. **Type-narrowing note:** `world.bios_chain` is typed `Mapping[str, BIOSChainEntry]` here while `resolve_bios_dependencies(bios_chain: dict[str, BIOSChainEntry])` is narrower. JobManager passes through a one-line `dict(world.bios_chain)` at the call site; widening the function signature itself would be a `copy/` change and is out of scope. |
| `file_started` | `{short_name, bytes_total}` | Synthesized: emitted on the first `on_progress(short, bytes_done, bytes_total)` tick for a new `short_name`. JobManager keeps a `_seen_shorts: set[str]` and emits when `short ∉ _seen_shorts`. |
| `file_progress` | `{short_name, bytes_done, bytes_total}` | Direct: every `on_progress` tick. The runner enables chunked progress whenever `on_progress is not None` (`copy/runner.py:263`), so a normal copy emits ~one tick per 1 MiB chunk via `_chunked_copy` (`copy/executor.py:26-37`); dry-run (`runner.py:259-260`) and idempotent-skip (`runner.py:294-295`) paths emit a single end-of-file tick. JobManager wires itself as `on_progress` unconditionally — there is no API knob to disable chunked progress in P04. |
| `file_finished` | `{short_name, bytes}` | Synthesized: emitted when an `on_progress` tick has `bytes_done == bytes_total` for the current `short_name`. (Both `runner.py:259` and `runner.py:295` end with this self-equal tick — see the explicit `on_progress(short, src.stat().st_size, src.stat().st_size)` calls — so this synthesis is well-defined.) |
| `paused` | `{at: datetime}` | Synthesized: at every `on_progress` tick, JobManager reads `controller.state` (existing public property on `CopyController`, lock-protected by the controller); emits `paused` on a `state != PAUSED` → `state == CopyControlState.PAUSED` transition. Polling at progress-tick frequency means resolution is "next file boundary" — matches `copy/spec.md` § "pause holds at file boundary." |
| `resumed` | `{at: datetime}` | Synthesized: same poll, `state == PAUSED` → `state != PAUSED` transition. |
| `job_finished` | `{job_id, status: CopyReportStatus, report_path: str}` | JobManager, after worker thread joins cleanly with a `CopyReport`. JobManager additionally writes the report to `data/copy-history/<job_id>/report.json` per § Atomic-write protocol before emitting. |
| `job_aborted` | `{job_id, reason: str, recycled_count: int}` | JobManager, after `controller.cancel(...)` plus worker join. `recycled_count` from `CopyReport.recycled` length (the FP05 A1 contract). |

The event stream **always ends** with exactly one of `job_finished` / `job_aborted`. The SSE handler closes the response after the terminal event. Playlist-conflict cancels (per R21) return 409 synchronously and **never become SSE events** — JobManager isn't created.

### Sync-to-async bridge

The worker thread runs `run_copy(plan, on_progress=...)`; the on-progress callback is the only sync→async surface. JobManager wraps it into the synthesizer described in the table above:

```python
from datetime import UTC, datetime                # Python 3.11+; project targets 3.12+

from mame_curator.copy import CopyControlState    # already exported from copy/__init__.py


class _ProgressSynthesizer:
    """Owned by JobManager; converts on_progress ticks into JobEvents.

    Runs on the worker thread (call sites in copy/runner.py); pushes events to
    the loop thread via call_soon_threadsafe (see below).
    """
    def __init__(self, loop: AbstractEventLoop, queue: asyncio.Queue[JobEvent],
                 controller: CopyController, job_id: str) -> None:
        self._loop = loop
        self._queue = queue
        self._controller = controller
        self._job_id = job_id
        self._seen: set[str] = set()
        self._was_paused = False

    def __call__(self, short: str, bytes_done: int, bytes_total: int) -> None:
        # 1. file_started on first sighting.
        if short not in self._seen:
            self._seen.add(short)
            self._emit("file_started", {"short_name": short, "bytes_total": bytes_total})
        # 2. file_progress always (the consumer can ignore if uninteresting).
        self._emit("file_progress",
                   {"short_name": short, "bytes_done": bytes_done, "bytes_total": bytes_total},
                   droppable=True)
        # 3. file_finished on size-equals-total tick.
        if bytes_done == bytes_total:
            self._emit("file_finished", {"short_name": short, "bytes": bytes_total})
        # 4. paused/resumed on controller-state transition. Read of `state` is
        #    lock-protected inside CopyController; safe to call from the worker thread.
        is_paused = self._controller.state == CopyControlState.PAUSED
        if is_paused != self._was_paused:
            self._emit("paused" if is_paused else "resumed", {"at": datetime.now(UTC)})
            self._was_paused = is_paused

    def _emit(self, event: str, payload: dict, *, droppable: bool = False) -> None:
        ev = JobEvent(event=event, payload=payload, ts=datetime.now(UTC))
        if droppable:
            self._loop.call_soon_threadsafe(_offer_droppable, self._queue, ev)
        else:
            # Guaranteed-delivery: schedule a put-coroutine on the loop, fire-and-forget.
            self._loop.call_soon_threadsafe(asyncio.create_task, self._queue.put(ev))
```

**Why `call_soon_threadsafe(asyncio.create_task, queue.put(ev))` and not `run_coroutine_threadsafe`.** `run_coroutine_threadsafe` returns a `concurrent.futures.Future` that the worker would have to `.result()` to await — but `.result()` blocks the worker on the loop thread, which is wasteful for a fire-and-forget put on a bounded queue. `call_soon_threadsafe(asyncio.create_task, coro)` schedules the coroutine on the loop thread without blocking the worker. `asyncio.create_task` on the loop thread is safe (we are inside the loop's call-soon handler when it executes). The queue's `.put()` will await if the queue is full — but the bound is 1024 and only non-progress events go through this path, so it never blocks in practice.

The droppable path uses a synchronous helper `_offer_droppable(queue, event)` that calls `queue.put_nowait(event)` and on `QueueFull` discards the oldest `file_progress` from the queue's `_queue` deque (only `file_progress` is droppable; queue size 1024, sized for ~20 files of headroom at 50 events/file under no-subscriber backpressure).

### Multi-subscriber fan-out

The `JobManager.events()` async iterator is consumable by **multiple** simultaneous clients (e.g. two browser tabs watching the same job). Implementation: each `events()` call gets its own `asyncio.Queue` (subscriber-local); the JobManager holds a list of active subscribers and broadcasts each emitted event to all of them. Backpressure on slow subscribers: drop-oldest on subscriber-local queues independently. The original "main" queue exists only as the producer-side staging point.

### Pause / resume / cancel

- `pause()` calls `Job.controller.pause()` (existing FP05 B4 logic — race-safe re-check).
- `resume()` calls `Job.controller.resume()`.
- `abort()` calls `Job.controller.cancel(recycle_partial=...)` (existing FP05 A1 logic — recycle-on-cancel).
- The route handlers wait 50 ms (`asyncio.sleep(0.05)`) for the controller flag to take effect, then return `JobStatus`. FP09 Cluster R Open-Q2 narrowed the prior "up to 250 ms" wording to match implementation — the controller flag is set synchronously inside `pause()` / `resume()` / `cancel()` (under the lock), so the wait is purely defense-in-depth against a CPU-bound worker that hasn't yet hit a `wait_if_paused()` boundary; 50 ms is enough headroom for the worker thread to pick up the flag in normal cases without making the API noticeably laggy.

## Filesystem sandbox + grant API

### Allowlist composition

At any moment the allowlist is:

```
allowed = {
    Path.home(),
    config.paths.source_roms,
    config.paths.source_dat.parent,
    config.paths.dest_roms,
    config.paths.retroarch_playlist.parent,
} | set(config.fs.granted_roots)
```

All entries are **resolved to absolute, symlink-followed canonical paths** (`Path.resolve(strict=False)`) at composition time. The composed allowlist is cached on `app.state.world` and recomputed whenever those config fields change.

### Path validation

Every `/api/fs/*` route that takes a `path` query parameter validates as follows (single helper `validate_fs_path` in `fs.py`):

1. Reject if `path` is empty or contains a NUL byte → 400.
2. `requested = Path(path).resolve(strict=False)` — symlinks resolved, `..` collapsed.
3. If no entry in the allowlist satisfies `requested.is_relative_to(root)` → 403 with body `{detail: "path outside allowlist", code: "fs_sandboxed"}`. The path is included in `detail` via `repr()` per FP06 B3.
4. If the route requires existence (`/api/fs/list`) and `requested.exists()` is False → 404.
5. If the route requires a directory (`/api/fs/list`) and `requested.is_dir()` is False → 400.

The validator is the **only** path-trust boundary. Once validated, handlers operate on `requested` directly.

### Grant API

`POST /api/fs/allowed-roots` body:

```python
class FsGrantRootRequest(BaseModel):
    path: str
```

Validation:

1. `path` resolves to an absolute existing directory → 400 if not.
2. `path` is not already covered by an existing allowlist entry (i.e. not `is_relative_to` any current root) → 409 with `{code: "fs_already_covered", existing_root: "..."}`.
3. Atomic-write `config.yaml` with the new root appended; snapshot prior; recompute `app.state.world`.

No "second-level" disk-root re-sandbox: the loopback bind + the user's own confirmation in the Settings UI are the trust boundary, and on Linux the disk-root check is a no-op anyway (`/` covers everything). If a future code path adds an alternate root source (network mount, etc.) and a hardening reason emerges, fold a second-level guard then.

`DELETE /api/fs/allowed-roots/{root_id}`:

1. Look up by `root_id` (a stable short-hash of the path; survives restarts).
2. 404 if unknown.
3. 400 if the root's `source == "config"` (not user-revocable; remove the corresponding `paths.*` config field instead).
4. Atomic-write + snapshot + recompute.

### Threat model

- **In-process traversal** (`../../etc/passwd`): blocked by `Path.resolve` + `is_relative_to`.
- **Symlink traversal** (a symlink inside `dest_roms` pointing to `/etc`): blocked because `Path.resolve(strict=False)` follows symlinks before the allowlist check.
- **TOCTOU between resolve and use**: not addressed — single-user local app, the user is the only writer. Documenting as accepted residual risk.
- **NUL byte injection**: blocked at step 1.
- **Network access via `/api/fs`**: bind is `127.0.0.1` by default; the loopback bind is the v1 trust boundary.

### Merge rules

Applied recursively per (key, value) in the body:

| Source-side type | Behavior |
|---|---|
| Scalar (`int`, `bool`, `str`, `Path`, enum) | Replace. |
| `tuple[T, ...]` (lists in YAML) | Replace (not extend). Documented explicitly so callers don't assume merge. |
| Nested model (`PathsConfig`, `ServerConfig`, ...) | Recurse into the section: each section's own field merge follows the same rules. |
| `None` (field absent) | No change to the existing value. |

Validation: build the full merged `AppConfig`, run Pydantic validation. On failure → 422 with body:

```python
{
    "detail": "config validation failed",
    "code": "config_invalid",
    "fields": [
        {"loc": "paths.source_roms", "msg": "path does not exist", "type": "path_not_found"},
        ...
    ],
}
```

### Path-existence validation

`paths.source_roms`, `paths.source_dat`, `paths.dest_roms` and `paths.retroarch_playlist`'s parent must exist (resolved). `media.cache_dir` is created if absent (idempotent `mkdir(parents=True, exist_ok=True)`). `fs.granted_roots` entries: each must exist (failure → 422).

### Side effects on success

1. Snapshot prior `config.yaml` to `data/snapshots/<timestamp>/config.yaml` (atomic write — see § Atomic-write protocol).
2. Atomic-write new `config.yaml`.
3. If any field on `filters:` (or its alias `picker:`) changed: rebuild `FilterContext`, re-run `run_filter`.
4. Swap `app.state.world` under the write-lock.
5. Return `AppConfig` (the new full config) with `restart_required: true` if `server.host` or `server.port` changed.

### Atomic-write protocol

Every YAML / JSON write the API performs (`config.yaml`, `overrides.yaml`, `sessions.yaml`, `data/notes.json`, `data/snapshots/<ts>/*`, `data/copy-history/<id>/report.json`) follows the same protocol — same discipline as `copy_one` in `copy/spec.md` § "Atomic copy primitive":

1. `tmp = dst.with_suffix(dst.suffix + ".tmp")` (same directory as `dst` so rename is intra-FS).
2. Open `tmp` for write; write bytes; `f.flush()`; `os.fsync(f.fileno())`; close.
3. `os.replace(tmp, dst)`.
4. `os.fsync` on the parent-directory fd (`os.open(dst.parent, O_RDONLY) → fsync → close`) so the rename is durable on power loss.
5. On any exception in 2–4: `tmp.unlink(missing_ok=True)`; re-raise wrapped in the appropriate `ApiException` subclass.

A single helper `atomic_write_bytes(dst: Path, data: bytes)` lives in a shared module (FP05 C2 already extracted `atomic_write_text` for `cli/`; the API helper extends that pattern to bytes for JSON-export paths). The CLI helper and the API helper share the same primitive — pure refactor moment.

### What does NOT trigger a re-run

- `ui.*` changes (purely client-side hints).
- `media.fetch_videos` (consumed only by media-fetch in P05+).
- `updates.*` (consumed only by `updates/` in P07+).
- `server.*` (requires restart).
- `fs.granted_roots` swaps (filter unchanged; sandbox does change).

This is checked by an explicit `_filter_relevant_changed(old, new) -> bool` helper to keep the recompute decision auditable.

## Media proxy (basic)

`GET /media/{name}/{kind}` ships a **minimal pass-through proxy** in P04. Implementation:

1. `{kind}` MUST be one of `boxart`, `title`, `snap`, `video`. Otherwise 400 (`code: "media_kind_invalid"`).

A single `app.exception_handler(ApiException)` formats the response body as `ApiErrorBody`. The naming distinction is deliberate: `ApiException` is the throwable, `ApiErrorBody` is the wire shape.

**No bare exception leaks**: a top-level `app.exception_handler(Exception)` catches anything else, returns 500 with `{detail: "internal error", code: "internal", fields: ()}`, logs the exception with traceback. The detail message NEVER includes the raw exception string (avoid information leak).

### Path interpolation

Every `detail` that includes a user-controlled string (paths, names, slugs) interpolates via `f"...{value!r}..."` per FP06 B3 / FP07 A4 / FP08 A1. Tests assert no literal LF survives in error responses (continuation of the FP06–FP08 pattern).

## Security

| Layer | Choice | Rationale |
|---|---|---|
| Bind | `127.0.0.1` default; `--host` override for advanced users | Single-user local tool; loopback eliminates network exposure. |
| Auth | None | Single-user local; loopback bind is the trust boundary. |
| CORS | Allowlist `http://localhost:5173` only (Vite dev) | Production same-origin; dev-server is the only legitimate cross-origin caller. |
| CSRF | Not implemented | Loopback + no auth + no cookies → no CSRF surface. |
| Path traversal | `Path.resolve` + `is_relative_to` allowlist | § Sandbox. |
| Job concurrency | Singleton with 409 on double-start | Eliminates whole class of race bugs; one user, one copy. |
| Error info leak | `detail` never includes raw exception text on 500 | Avoid leaking implementation details. |
| Markdown HTML | `markdown-it-py` `html: false` | Renders no inline HTML; XSS surface zero. |

These are **explicit v1 choices**, not gaps. A `docs/decisions/0004-api-security-model.md` ADR will record them at P04 close.
