# P05 — Media subsystem (`media/`) spec

**Status:** in-flight (App-Build Step 1 — spec for sign-off).
**Companion docs:** [Design §6.3](../superpowers/specs/2026-04-27-mame-curator-design.md) · [Long-form roadmap Phase 5](../superpowers/specs/2026-04-27-roadmap.md) · [Coding standards](../standards/coding-standards.md).

## Contract

Given a parsed DAT, this module produces verified media URLs for any `Machine` and lazily caches fetched bytes to disk. Specifically:

1. **Build URLs** to libretro-thumbnails images (boxart / title / snap) from a `Machine`, applying libretro's filename escape rules verbatim so a description like `Q*bert's Qubes` resolves to the correct file.
2. **Fetch with cache** — `async fetch_with_cache(url, cache_dir, *, client)` returns the on-disk path if the URL is already cached; otherwise downloads via the lifespan-managed `httpx.AsyncClient`, writes atomically, and returns the new path. 404 is the no-image sentinel; other upstream errors raise.
3. **Wire R39** — `routes/media.py` swaps its inline URL build + bare `client.get` for `urls_for` + `fetch_with_cache` so subsequent requests for the same image hit disk, not the network.

The module depends on `parser/` (for `Machine`) and `_atomic.atomic_write_bytes`. It MUST NOT import from `api/`, `filter/`, `copy/`, `updates/`, `setup/`, or `help/` per the project's anti-jump rule. The R39 wiring lives in `api/routes/media.py` (caller side); `media/` itself stays HTTP-agnostic.

## Public API

### `escape_libretro(name: str) -> str`

Mirrors the libretro-thumbnails filename rule for the [MAME repo](https://github.com/libretro-thumbnails/MAME): every occurrence of any character in the set `& * / : \ < > ? | "` (10 chars: ampersand, asterisk, slash, colon, backslash, less-than, greater-than, question-mark, pipe, double-quote) is replaced with `_`. Apostrophes (`'`), spaces, parentheses, brackets, periods, hyphens, and unicode letters pass through unchanged.

Design §6.4 line 732 and roadmap line 396 list the same 10 characters, but the markdown rendering breaks across an unbalanced backtick — the **intended** character set is the canonical 10 above, which P05 follows. The malformed markdown is a doc-rendering bug worth tidying in a future doc-sweep but does not change the contract.

| Input | Output |
|---|---|
| `Q*bert's Qubes` | `Q_bert's Qubes` |
| `Tron / Discs of Tron` | `Tron _ Discs of Tron` |
| `Pac-Man` | `Pac-Man` |
| `Robotron: 2084` | `Robotron_ 2084` |
| `Foo \| Bar` | `Foo _ Bar` |

The escape is applied **before** percent-encoding via `urllib.parse.quote`. A description containing `%` or `#` would still need percent-encoding (those are URL-reserved); the libretro escape only handles filename-illegal chars on Windows. Order: escape → quote → concat into URL.

### `class MediaUrls` (frozen Pydantic)

```python
class MediaUrls(BaseModel):
    model_config = ConfigDict(frozen=True, extra="forbid")
    boxart: str
    title: str
    snap: str
```

Three URLs per machine. Video is intentionally absent — design §6.3 routes video through progettoSnaps, not libretro-thumbnails, and that's a P06+ Settings-page-toggle concern (long-form roadmap Phase 5 "Out of scope"). **Load-bearing consequence:** `getattr(MediaUrls, "video")` raises `AttributeError`, so any caller dispatching on the `kind` query parameter MUST short-circuit on `kind == "video"` BEFORE calling `urls_for(...)`. The R39 wiring example below honours this.

### `urls_for(machine: Machine) -> MediaUrls`

Builds all three URLs for one machine. Pseudocode:

```python
def urls_for(machine: Machine) -> MediaUrls:
    escaped = escape_libretro(machine.description)
    encoded = quote(escaped)
    return MediaUrls(
        boxart=f"{_BASE_URL}/Named_Boxarts/{encoded}.png",
        title=f"{_BASE_URL}/Named_Titles/{encoded}.png",
        snap=f"{_BASE_URL}/Named_Snaps/{encoded}.png",
    )
```

`_BASE_URL = "https://raw.githubusercontent.com/libretro-thumbnails/MAME/master"` (string-equal to the constant inlined in `routes/media.py` today; P05 moves it to `media/urls.py` and the route imports it).

### `cache_path_for(url: str, cache_dir: Path) -> Path`

Returns `cache_dir / f"{sha256(url.encode('utf-8')).hexdigest()}{ext}"` where `ext` is the URL path's suffix (`.png` for libretro thumbnails). Pure function — no I/O. Used by `fetch_with_cache` and by callers who want to probe cache existence without triggering a download.

### `async fetch_with_cache(url, cache_dir, *, client) -> Path | None`

Signature:

```python
async def fetch_with_cache(
    url: str,
    cache_dir: Path,
    *,
    client: httpx.AsyncClient,
) -> Path | None: ...
```

Behavior:

1. Compute `path = cache_path_for(url, cache_dir)`.
2. If `path.exists()` → return `path` (no network hit).
3. `resp = await client.get(url)` — relies on the caller-supplied `httpx.AsyncClient`'s default timeout. No `timeout=` parameter on `fetch_with_cache` keeps the helper's surface minimal and prevents two callers from picking different defaults — global rule 2 (shortest correct implementation). **Implementation prerequisite (one-line code change):** the lifespan-managed client at `api/app.py:40` is currently constructed as bare `httpx.AsyncClient()` — falling back to httpx's 5s library default. P05 changes this to `httpx.AsyncClient(timeout=10.0)` so the helper preserves P04's effective per-request timeout (`routes/media.py:48` had an inline `timeout=10.0` per call). The 10s value moves from per-call to client construction; functional behavior unchanged.
4. `resp.status_code == 404` → return `None` (sentinel: no image upstream; do not cache the negative result).
5. `resp.status_code != 200` → raise `MediaFetchError(f"upstream {resp.status_code} for {url!r}")`.
6. Otherwise: `atomic_write_bytes(path, resp.content)` and return `path`.
7. `httpx.HTTPError` from step 3 (timeout, network) → re-raise as `MediaFetchError(...)` chained via `from exc`.

Concurrency note: two concurrent calls for the same URL both hit step 3 (existence check is racy). Both writes go through `atomic_write_bytes`, which uses unique `.tmp` names + `os.replace`, so the cache file is never torn. Both calls return the same final path. The wasted second download is acceptable — pre-warming a per-URL `asyncio.Lock` would couple `media/` to the event loop in a way that doesn't pay off at single-user scale.

### Exceptions

```python
class MediaError(Exception):
    """Base class for media subsystem errors."""

class MediaFetchError(MediaError):
    """Upstream non-200 (other than 404) or network failure."""
```

The R39 route catches `MediaFetchError` and translates to `MediaUpstreamError(502)`. A `None` return from `fetch_with_cache` translates to `MediaUpstreamNotFoundError(404)`. The route layer keeps owning HTTP status mapping; `media/` raises typed Python errors.

## Cache layout

- **Directory source:** `media/` itself doesn't read config — it takes `cache_dir: Path`. The R39 route reads `app.state.world.config.media.cache_dir` (existing `MediaConfig.cache_dir` field shipped in P04 — `api/schemas.py:69`, default `./data/media-cache/`) and passes it in. **No `AppConfig` schema change needed.**
- **Filename:** `<sha256(url).hexdigest()><ext>` where `ext = Path(urlparse(url).path).suffix` (always `.png` for libretro). Example: `abc123…def.png` for `https://…/Named_Boxarts/Pac-Man.png`. URLs without a path suffix produce a bare-hex filename — see test `test_cache_path_for_no_url_suffix`.
- **Subdirectories:** none. Flat. SHA-256 hex collision space is large enough that 100k images is fine in one directory on every supported filesystem (ext4, NTFS, APFS).
- **Write protocol:** unique `.tmp` sibling → `os.replace` → fsync parent dir. Inherited from `_atomic.atomic_write_bytes`; no media-specific path. The tmp file is created **in the same directory** as the final cache file (not in `tempfile.gettempdir()`), so `os.replace` is always in-filesystem and EXDEV doesn't apply.
- **Read protocol:** `Path.exists()` + return path. No mtime check, no checksum, no LRU. The cache is permanent by design (design §6.3) — clearing is a Settings-page action wired in P06+.

## R39 wiring

`src/mame_curator/api/routes/media.py` after P05:

```python
from mame_curator.media import MediaFetchError, fetch_with_cache, urls_for

@router.get("/media/{name}/{kind}")
async def media_proxy(
    name: str, kind: str, request: Request, world: WorldState = Depends(get_world)
) -> Response:
    if kind not in {"boxart", "title", "snap", "video"}:
        raise MediaKindInvalidError(f"unknown media kind: {kind!r}")
    machine = world.machines.get(name)
    if machine is None:
        raise GameNotFoundError(f"game not found: {name!r}")
    if kind == "video":
        # Load-bearing: MediaUrls has no `video` field, so getattr below would
        # AttributeError. Also reflects design §6.3 — video routes through
        # progettoSnaps (P06+), not libretro-thumbnails.
        raise MediaUpstreamNotFoundError(f"video upstream not configured for {name!r}")
    urls = urls_for(machine)
    url = getattr(urls, kind)
    cache_dir = world.config.media.cache_dir
    client: httpx.AsyncClient = request.app.state.media_client
    try:
        path = await fetch_with_cache(url, cache_dir, client=client)
    except MediaFetchError as exc:
        raise MediaUpstreamError(f"upstream error: {exc!r}") from exc
    if path is None:
        raise MediaUpstreamNotFoundError(f"upstream 404 for {url!r}")
    return Response(content=path.read_bytes(), media_type="image/png")
```

The handler stays under 30 lines. `path.read_bytes()` is a sync read inside an async handler — justified because libretro-thumbnails PNGs are <500 kB typical, <2 MB worst-case, and the cache lives on local disk. If video thumbnails (P06+, ~30 MB) ever route through this code path, switch to `FileResponse(path)` or `aiofiles` streaming. `MediaUrls` has no `video` field today, so this is not a current risk.

No `AppConfig` schema change. `MediaConfig.cache_dir` already exists from P04 (`api/schemas.py:69`).

## Out of scope (deferred)

- **Video thumbnails.** Design §6.3 routes video through progettoSnaps; that's P06+ when the Settings page wires the opt-in. P05's R39 returns `media_upstream_not_found` for `kind=video`.
- **Cache eviction / size cap.** Permanent by design (§6.3). Settings-page "Clear cache" button is P06+.
- **Mass pre-fetching.** Always lazy. The frontend's `<img src>` populates the cache organically as the user browses.
- **Content-type sniffing.** Trust the URL suffix; libretro-thumbnails serves `.png` exclusively.
- **Per-URL fetch deduplication.** Wasted second downloads on concurrent same-URL requests are acceptable at single-user scale.
- **Cache key versioning.** A description change (DAT bump) → new URL → new cache entry; old entry remains. By design — readers on the old DAT still hit their cache.

## Architecture notes

- **Module is HTTP-agnostic.** `media/` knows nothing about FastAPI, `Request`, or status codes. Callers pass in the `httpx.AsyncClient` (lifespan-managed) and translate exceptions. This keeps unit tests pure and lets a future CLI consumer reuse the helper.
- **Atomic-write reuse.** `_atomic.atomic_write_bytes` is already used in `api/persist.py` (4 sites) and `api/jobs.py` (1 site). P05 is one more site, well past Rule of Three — just import.
- **No retry policy.** httpx's default of zero retries propagates. If we add retry later, it sits at the route layer (HTTP semantics) not the cache layer (storage semantics).
- **No async lock around the cache.** See concurrency note in `fetch_with_cache`.
