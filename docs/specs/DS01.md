## Why a single fix-pass and not three

The findings cluster into three groups, but they share a single
risk: **silent drift between spec, code, and CHANGELOG**. Closing
them piecemeal would mean three separate `/audit` + `/indie-review`
gates and three commits that each leave the project in a weirdly
half-aligned state. One fix-pass with three thematic sub-blocks
keeps the audit surface coherent and lets the closing review check
spec/code/record alignment in a single pass.

### Cluster B — Test gaps (FP01-deferred)

- **B1** — Hypothesis property tests for `resolve_bios_dependencies` (verified to exist at `copy/bios.py:12`). Properties to encode:
  1. **Idempotence:** running `resolve` on its own output's BIOS set ∪ original winners yields the same BIOS set + warnings.
  2. **Transitive reachability:** every name in the returned BIOS set is reachable from at least one winner via a finite chain of `romof` / `<biosset>` edges (no orphan emissions).
  3. **Winner exclusion:** the returned BIOS set is disjoint from the input winner set (`bios.py:37, 42` enforce this — property locks it as a contract).
  4. **Order independence:** `resolve({a, b}, chain) == resolve({b, a}, chain)` (sorted-tuple equality on warnings; set equality on BIOS).
- **B2** — `test_cancel_with_keep_partial` strengthening. Current test (`tests/copy/test_runner.py:308-326`) only asserts cancel-before-start. Add a second test that:
  1. Stages two winners. The default `source_dir` fixture writes ~600-byte zips (`name.encode() * 100`); `executor._CHUNK` is 1 MiB, so the default fixture **never** triggers `_chunked_copy`. The new test must either (a) use a per-test fixture that writes ≥ 1 MiB zips so `_chunked_copy` fires (preferred — exercises the actually-cancellable code path), or (b) explicitly accept that the cancel trigger fires between files via the progress callback's *call boundary*, not mid-chunk. Choose (a) and document the fixture-size choice inline.
  2. Sets `recycle_partial=False`.
  3. Cancels via `controller.cancel(recycle_partial=False)` after the *first* winner finishes copying (use a progress callback as the cancel trigger).
  4. Asserts the first winner's `dst` survives and is intact (size + content); the second winner's `dst` does not exist.
- **B3** — `test_lpl_no_bom` strengthening. Current test (`tests/copy/test_playlist.py:104-110`) asserts `not raw.startswith(b"\xef\xbb\xbf")`. Strengthen to: read the file as raw bytes, decode as UTF-8 strict, round-trip through `json.loads`, assert the decoded payload equals what `write_lpl` accepted. This catches BOM-emission *and* any future encoding regression.
- **B4** — Widen `source_dir` fixture (`tests/copy/conftest.py:62-71`) from function scope to `scope="module"`. Cold-eyes review of all `source_dir` usages in `tests/copy/` (`test_executor.py`, `test_fp02_fixes.py`, etc.) confirmed every consumer reads only — none mutate the fixture's contents (mutations like `monkeypatch.setattr` target `shutil`/runner internals, not the fixture directory). Module scope eliminates the per-test rebuild cost.

## Architecture notes

This fix-pass does **not** change any public API surface. The only model-shape changes are:

- `BIOSResolutionWarning.kind` — `Literal[...]` arm narrows from 2 to 1 (drop `self_reference`). Pre-v1 so no compat shim needed.
- `FilterResult.dropped` — type changes from `dict[str, DroppedReason]` to `tuple[tuple[str, DroppedReason], ...]`. Subscript reads (`.dropped["key"]`) break under tuple shape and are rewritten per C2's enumerated read-site list. `len(.dropped)` survives unchanged. Pre-v1 so no compat shim needed.
- `AppendDecision` — unchanged from FP02.

The CLI argument surface is unchanged — every flag already has a `help=` string at HEAD (the original pre-P03 sweep finding was stale, see C9 footnote). The report.json path becomes atomic; the sentinel-path quirk goes away. No user-visible behavior change beyond clearer errors and more durable tracebacks (A5 surfaces stack frames that were previously swallowed).
