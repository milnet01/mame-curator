# P06 — Frontend MVP (`frontend/`) spec

**Status:** in-flight (App-Build Step 1 — spec for sign-off).
**Companion docs:** [Design §8](../superpowers/specs/2026-04-27-mame-curator-design.md) · [Long-form roadmap Phase 6](../superpowers/specs/2026-04-27-roadmap.md) · [Coding standards](../standards/coding-standards.md).

## Contract

A browser UI that consumes the P04 HTTP API (40 routes) and the P05 media proxy (R39) to deliver every interactive flow named in design §8: browse the library in four selectable layouts, drill into a game's alternatives + override flow, run dry-run + live copy with SSE progress, manage sessions, browse activity + stats, and edit Settings. Built with Vite + React 19 + TypeScript + Tailwind v4 + shadcn/ui, served as static assets from `frontend/dist/` mounted at `/` on the FastAPI app. **No setup wizard** (Phase 8). **No app self-update / INI refresh wiring** (Phase 7) beyond the layout shell. **No client-side filesystem access** — every disk operation crosses the API per long-form roadmap "Things this project deliberately does not do".

The deliverable is the running UI plus its production bundle (`frontend/dist/`), committed so end users running `git clone && ./run.sh` reach a working app without installing Node.

## API contract surface

The frontend reads the FastAPI app at the same origin (after the static mount in §"Static-file serving" lands). Dev mode runs the Vite dev server on `http://localhost:5173` (Vite's default; matches P04's CORS allowlist at `docs/specs/P04.md:867`) and proxies `/api/*` and `/media/*` to `http://127.0.0.1:8080`. Production mode is same-origin via the static mount; CORS does not engage.

**Type-mirroring strategy: hand-mirrored** in `frontend/src/api/types.ts`, per `docs/standards/coding-standards.md` § 13 ("auto-generated TS types via `pydantic-to-typescript` or hand-mirrored `interface`s with a CI check that they match"). The CI gate `tools/check_api_types_sync.py` parses both Pydantic models from `mame_curator.api.schemas` and TS interfaces from `frontend/src/api/types.ts` and **fails if any field is present in either side and absent from the other** — bidirectional parity, not one-way drift. (The bidirectional rule is what closes the silent-drop class of bug: Pydantic adds a field, TS doesn't mirror it, runtime drops the field, no test catches it.) Reason for hand-mirrored over `pydantic-to-typescript`: only ~25 of the API's ~60 schema types are consumed client-side, so writing them by hand is less code than wiring + auditing a generator's output. If the generator ever becomes desirable (e.g. P07 adds 30+ new types), revisit.

Beyond field parity, `client.ts` MUST validate every response with a strict parser that rejects unknown fields (mirroring Pydantic's `extra="forbid"`). Implementation: a small `parse<T>(schema, data)` helper using `zod` (`^4.x`; the v4 wire format is stable and bundle weight stays under the 350 kB gzipped budget enforced in §"Architecture notes") — alternative to hand-rolling `assert no extra keys`. Adding `zod` to dev+runtime deps is allowed under the toolchain table.

### Error envelope handling

The API renders typed errors as `ApiErrorBody { detail: str, code: str, fields?: FieldError[] }` (`api/errors.py:31`). The frontend's `api/client.ts` wrapper:

1. Reads `response.json()` on non-2xx, validates the `ApiErrorBody` shape.
2. Throws an `ApiError` JS class carrying `code`, `detail`, `fields`.
3. React-query's `onError` surfaces user-facing messages from `strings.ts` keyed by `code`; falls back to `detail` for unknown codes.

Codes the UI must handle distinctly (others fall to a generic toast). Mirrors the typed-error classes in `src/mame_curator/api/errors.py` — keep these in lockstep:
`config_invalid`, `game_not_found`, `override_not_found`, `session_not_found`, `session_name_invalid`, `fs_sandboxed`, `fs_already_covered`, `fs_path_invalid`, `fs_not_found`, `fs_root_not_found`, `fs_config_root_not_revocable`, `job_already_running`, `job_not_found`, `copy_report_corrupt`, `playlist_conflict_cancelled`, `snapshot_not_found`, `help_topic_not_found`, `media_kind_invalid`, `media_upstream_error`, `media_upstream_not_found`, `validation_error`.

### SSE consumer (R25)

`useCopyJob` opens an `EventSource('/api/copy/status')` **after R21 returns `JobAccepted{job_id}`** (never speculatively — R25 returns 404 when no active job exists, and we don't want a transient 404 on every page mount). The hook parses each `JobEvent` payload and reduces it into render state: `{ files_done, files_total, bytes_done, bytes_total, current_file, state, recent_warnings[] }`. Pause/resume/abort POST to R22/R23/R24. The hook auto-reconnects on transient stream loss but **does not** retry on `JobNotFoundError` (no current job → close stream cleanly). On page load with no job state in memory but a still-running job server-side, the hook discovers the running job by re-trying R25 once and reads `state` from the first SSE event — covers the "user reloaded mid-copy" path.

## Static-file serving (backend addition)

`api/app.py` gains one mount, conditional on `frontend/dist/` existing. The mount uses a thin `_SPAStaticFiles(StaticFiles)` subclass so deep SPA links resolve while typo'd API/media/asset paths still surface as a real 404:

```python
from starlette.staticfiles import StaticFiles

# `src/mame_curator/api/app.py` → parents[3] = repo root.
_FRONTEND_DIST = Path(__file__).resolve().parents[3] / "frontend" / "dist"


class _SPAStaticFiles(StaticFiles):
    """`StaticFiles` that serves `index.html` for path-shaped 404s only.

    Vanilla `StaticFiles(html=True)` only serves `index.html` when the
    request resolves to a directory; deep SPA links like
    `/sessions/foo` return 404. The carve-outs (`api/`, `media/`,
    `assets/`) MUST NOT cascade to `index.html` — typo'd API paths
    must surface as a real 404 so the frontend's zod validator
    distinguishes "missing route" from "shape mismatch", and a
    missing asset must 404 visibly so it doesn't get parsed as JS.
    """

    _NO_FALLBACK_PREFIXES = ("api/", "media/", "assets/")

    async def get_response(self, path, scope):
        try:
            return await super().get_response(path, scope)
        except StarletteHTTPException as exc:
            if exc.status_code != 404:
                raise
            if path.startswith(self._NO_FALLBACK_PREFIXES):
                raise
            return await super().get_response("index.html", scope)


if _FRONTEND_DIST.is_dir():
    app.mount("/", _SPAStaticFiles(directory=_FRONTEND_DIST, html=True), name="frontend")
```

Mounted **after** `app.include_router(api_router)` so `/api/*` and `/media/*` keep precedence at the FastAPI router level; the `_NO_FALLBACK_PREFIXES` carve-out is a defence-in-depth layer for paths the static mount itself receives (typo'd routes that don't match any FastAPI route fall through to the mount). If `frontend/dist/` is absent (dev mode), the mount is skipped silently and the user runs Vite on `:5173` against the API on `:8080` — the dev experience.

This is the **only** backend code change in P06. No new API routes. No schema changes. No new dependencies on the Python side.

## Component contracts (binding details only)

- **`GameCard`** — props: `card: GameCard, focused?: boolean, onOpen: () => void`. Renders flyer via `/media/{name}/boxart` with `onError` fallback to placeholder SVG. Shows corner badges from `card.badges` per design §8 mapping. **The frontend MUST NOT request `/media/{name}/video`** — R39 short-circuits `kind=video` to `media_upstream_not_found` (404) by design, and `MediaUrls` has no `video` field upstream; P06 only renders `boxart` (cards) and `boxart`+`title`+`snap` (detail drawer). The `media.fetch_videos` Settings toggle is forward-wiring for a post-P06 progettoSnaps integration; flipping it has no observable client behavior in P06.
- **`LibraryGrid`** — props: `cards: GameCard[], layout: "masonry" | "list" | "covers" | "grouped", groupKey?: "genre" | "year" | "publisher"`. Virtualizes via `@tanstack/react-virtual`. Acceptance: 3,000-card fixture renders in `<200ms` first paint and scrolls at 60 fps in dev tools.
- **`FiltersSidebar`** — every binary preference uses `<Switch>` from shadcn (NOT `<Checkbox>`). Year-range slider, free-text search (debounced 200 ms), Save-as-session button.
- **`AlternativesDrawer`** — slides from the right, shows parent + clones with media, click-to-override calls R11 then closes. `WhyPickedPanel` underneath renders the `Explanation.hits` chain.
- **`ActionBar`** — sticky bottom; "N games · X.Y GB · Z BIOS deps · [Dry-run] [Copy]". Counts pulled from `useGames` aggregate + a small client-side BIOS-dep tally (data already in `GameDetail`).
- **`CopyModal`** — consumes `useCopyJob` SSE state; pause/resume/cancel buttons; conflict prompts surface as inline panels whose Resolve buttons call R21 with `append_decisions`. Dismissible during copy (work continues; status indicator stays in `AppShell` top bar).
- **`ConfirmationDialog`** — wrapper around `<AlertDialog>` enforcing the design §8 rule: destructive-action label is concrete (`Delete 3 files from drive`), never `"OK"`. Used by: clear cache, restore snapshot, reset config, abort-with-recycle, override-replacing-existing.
- **`CmdKPalette`** — `<Command>` from shadcn; sources: games (filtered list), settings sections, navigation actions, help topics. Phase-7 self-update commands are explicitly NOT registered here.
- **`ThemeSwitcher`** / **`LayoutSwitcher`** — dropdowns; persist via R15 PATCH to `ui.theme` / `ui.layout` and refetch R14 (which carries `restart_required: false` for these).
- **`SettingsPage` cards-per-row hint.** `UiConfig.cards_per_row_hint` (`Literal["auto", 4, 5, 6, 8]`, `api/schemas.py:78`) is **YAML-only in P06** — no Settings control. The grid honours the value when present (passed via `useConfig`), defaults to `auto`. The Phase-7+ Settings expansion adds a UI affordance.

### State management

- **Server state:** react-query, default `staleTime: 30_000`. The library grid query has `staleTime: 5_000` to feel snappy after filter changes.
- **Local UI state:** React `useState` / `useReducer`. No Zustand / Redux / Jotai — react-query covers the server-state half and the rest is small enough for component state + a handful of context providers (theme, keyboard registry).
- **Persisted user state:** lives in `config.yaml` server-side via R15 PATCH (theme, layout, default sort, etc.) — not in `localStorage`. Reason: a user opening the app from a second browser sees their preferences, and the wizard's `config.yaml` export bundles them.

## Themes

Six palettes per design §8.5: `dark` (default), `light`, `double_dragon`, `pacman`, `sf2`, `neogeo`. Each is a `@theme` block in `index.css` keyed on a `data-theme="…"` attribute on `<html>`. `ThemeSwitcher` toggles the attribute; CSS variables propagate to every shadcn primitive automatically. Implementation rule: a theme adds **only** color variables — no per-theme assets, no per-theme JS, no per-theme component variants. Inter font (system fallback) across all themes.

## Implementation steps (strictly ordered)

Re-ordered from the long-form roadmap's 28-step list to surface the type-sync CI gate early (step 7) and merge the per-component TDD steps into a single ordered loop (step 10).

1. Scaffold `frontend/` via `npm create vite@latest -- --template react-ts`. Commit the scaffold + `tsconfig.json` + bare `App.tsx`.
2. Install Tailwind v4 + `@tailwindcss/vite`. Verify `index.css` has `@theme`. Define palette variables for the six themes per design §8.5.
3. Initialize shadcn/ui: `npx shadcn@latest init`. Add the primitives listed in design §8.5.
4. Install `@tanstack/react-query`, `@tanstack/react-virtual`, `react-router`, `lucide-react`, `framer-motion`, `clsx`, `tailwind-merge`.
5. Install dev deps: `vitest`, `@testing-library/*`, `jsdom`, `msw`, `@playwright/test`. Wire `vitest.config.ts` and `playwright.config.ts`.
6. Write `frontend/src/api/types.ts` (hand-mirrored from `mame_curator.api.schemas`) and `frontend/src/api/client.ts` (fetch wrapper + `ApiError`).
7. Write `tools/check_api_types_sync.py` and run it. Commit.
8. Write `frontend/src/strings.ts` with the user-facing strings catalogue (section-keyed).
9. Add the backend static mount in `api/app.py` (the §"Static-file serving" snippet). Add a backend test that asserts `app.mount("/", …)` is registered when `frontend/dist/` exists and absent when it doesn't. Smoke: build a stub `frontend/dist/index.html`, hit `GET /`, assert 200 + body match.

## Architecture notes

- **Frontend imports nothing from Python** at runtime. Type parity is enforced by the CI script, not by import. This keeps the build a pure `npm run build`.
- **No client-side filesystem access.** Every disk operation goes through R29–R34. The Settings → Paths section uses `<FsBrowser>` (built on R29 with R30/R31 for starting nodes) for path pickers; the browser never sees `window.showDirectoryPicker` or similar.
- **No third-party CDN.** All assets bundle into `frontend/dist/`. Themes are CSS variables — no Google Fonts call, no remote stylesheet.
- **Bundle weight budget.** Initial JS budget ≤ 350 kB gzipped, enforced by a CI gate using `size-limit` (`size-limit` reads thresholds from `package.json` `"size-limit"` array; failing build fails CI). Per-route lazy imports for `SessionsPage`, `ActivityPage`, `StatsPage`, `SettingsPage`, `HelpPage` (`React.lazy` + Suspense with shell skeleton). The Library bundle stays in the entry chunk because it's the landing page.
- **Accessibility.** shadcn primitives ship with Radix accessibility baked in; every interactive element gets a recognizable accessible name from `strings.ts`. Lighthouse ≥ 90 is the gate; we do not aim for AA-perfect WCAG until P09 polish.

## Anti-jump compliance

- No imports from `updates/`, `setup/`, or `help/` content beyond reading their existing P04 stub routes (R35–R38). Specifically:
  - Help page reads R37 + R38 — no markdown-rendering library bundled in P06; the API returns rendered HTML in `HelpContent.html`.
  - Updates banner reads R36 — no comparison logic client-side.
  - Setup banner reads R35 — no wizard.
- No new Python runtime deps. (`pydantic-to-typescript` would be one — explicitly rejected in §"API contract surface".)
- No new Python dev deps. The new CI gate `tools/check_api_types_sync.py` uses stdlib only (`ast`, `pathlib`, `re`).
- **Touched Python files:** `src/mame_curator/api/app.py` (one mount block), plus the new `tools/check_api_types_sync.py`. Nothing else in `src/mame_curator/` changes.
