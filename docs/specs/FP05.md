All findings share two themes: (1) **immutability invariants** that DS01 fixed for `FilterResult.dropped` but didn't propagate to `FilterContext` and the `BIOSResolutionWarning.kind` Literal; (2) **error-path completeness** — the C6+C7 patches in DS01 were narrow first cuts that the closing review showed needed broader application. Closing them piecemeal would mean three separate `/audit` + `/indie-review` cycles; one fix-pass with three thematic tiers keeps the audit surface coherent.

## Scope

### Tier 1 — real bugs

- **A1** — Implement `recycle_partial=True` cancellation. `copy/spec.md` § Pause/Resume/Cancel contracts: cancel with `recycle_partial=True` "every successfully-copied file from the current session is moved to recycle (via `recycle_file(reason="CANCELLED_RECYCLE_PARTIAL")`); report records the recycled set; one `copy_aborted` activity event with `details.recycled_count = N`." **Both winner-role and bios-role outcomes** count — the spec wording is "every file from the current session," and the recycle bin is keyed on session_id so the user can restore a partial run end-to-end. `copy/runner.py:305` reads `cancelled_mid = ctl.should_cancel()` and skips the playlist write but never inspects `controller.recycle_partial`. **Fix:** between the cancelled_mid check and the activity-log emission, walk every entry in `succeeded` (winner + bios), call `recycle_file(outcome.dst, reason="CANCELLED_RECYCLE_PARTIAL", session_id=session_id, recycle_root=plan.dest_dir / "data" / "recycle")` for each, accumulate `RecycleRecord`s into `report.recycled`, and emit the `copy_aborted` event with the count. The `CopyAbortedDetails.recycled_count` field already exists.
- **A2** — Reject empty-string `active` and empty session keys. Two related fixes in `filter/sessions.py`:
  1. In `Sessions._active_must_reference_a_defined_session`, additionally reject `self.active == ""` with a `SessionsError` (currently empty-string passes the `is not None` check and any `""` key in `sessions` matches).
  2. In `from_raw` / `load_sessions`, reject empty-string keys in the `sessions:` mapping with a `SessionsError`.
- **A3** — Narrow `copy/runner.py:259` `except Exception` to `except (OSError, CopyError)`. `CopyError` is the project's base class for every copy-module error (`copy/errors.py:1-7`); `CopyExecutionError(CopyError)` is `copy_one`'s wrapped `OSError`. Catching `CopyError` covers the documented failure mode; the additional `OSError` is defense-in-depth in case a fresh OSError escapes the `copy_one` wrapping (line-by-line review of `executor.py` shows `OSError` is wrapped, so the OSError clause is theoretical — but cheap to keep). The bare `Exception` is wider than spec asks for: `MemoryError` (subclass of `Exception`), `RecursionError` (also `Exception`), and any future custom non-OSError from `shutil.copy2` would all currently be swallowed into `report.failed` and the loop would continue after OOM — exactly wrong. `KeyboardInterrupt` and `SystemExit` derive from `BaseException` so they propagate either way. The `logger.exception` (DS01 A5) survives — it just logs less.

### Tier 3 — refactors

- **C1** — Extract `_read_capped_text(path, *, exc_cls)` helper. Currently duplicated across `filter/sessions.py:67-86` and `filter/overrides.py:30-49`, differing only in which exception class to raise. Put it in `filter/_io.py` (project-internal). Combine with B5: single open-stat-read implementation, one place to bump the cap, no skew risk.
- **C2** — Extract `atomic_write_text(path, text)` helper. Currently `cli/__init__.py:189-203` and `copy/executor.py:60-79` (binary copy) both do the `.tmp` + `os.replace` + try/finally cleanup dance. Two correct copies of a 6-line idiom is the threshold (per Rule of Three's intent — extract on third site, but two near-identical implementations that have to track each other is the warning sign). **Implementation:** use `tempfile.NamedTemporaryFile(dir=path.parent, prefix=path.name + ".", suffix=".tmp", delete=False)` for a unique tmp-name (defends against concurrent invocations and stale `.tmp` from prior crashes). **Cross-platform note:** on Windows, an open file handle prevents `os.replace` over the same path — the helper must `close()` the tmp before the `replace` step, not relying on a context-manager exit (CPython on Linux is forgiving here, Windows is not). Put in `mame_curator/_atomic.py` (cross-module helper). The CLI write becomes a one-liner; `executor.py`'s binary copy keeps its inline form (different shape — it copies bytes via `_chunked_copy`, not write-text).
- **C3** — Document `os.replace` cross-filesystem hazard. Wrap in `try/except OSError as exc` with `if exc.errno == errno.EXDEV: raise <typed-error>(...) from exc`. Theoretical risk (would require `args.out` resolving via symlink across mount points) but the typed error is one line and surfaces the failure mode cleanly. Apply to both `cli/__init__.py` (post-C2 helper) and `copy/executor.py:72`.

## Architecture notes

- `BIOSResolutionWarning.kind` returns to a 2-arm `Literal` with semantic content for both arms (B1); a new optional `via_winner: str | None = None` field threads the originating-winner through transitive misses. **`via_winner` semantics:** `None` for `kind="missing_from_listxml"` (the missing `name` *is* the top-level winner — no chain traversed); set to the originating winner short-name for `kind="transitive_missing_in_chain"` (the missing `name` was reached via a chain rooted at `via_winner`).
- `FilterContext` keeps dict types for hot-path performance; B6 ships a class-level docstring + per-field convention comment, **not** a runtime `MappingProxyType` wrapper. Future fix-pass can revisit if a real footgun surfaces; the round-2 cold-eyes note flagged that comment-only is enforcement-light, but a Pydantic v2 round-trip-safe `MappingProxyType` migration needs both `BeforeValidator` and `PlainSerializer` and that's deferred.
- `filter/_io.py` and `mame_curator/_atomic.py` are new project-internal modules (C1, C2). Both are dependency-free utility modules; neither violates the layer order.
- `cli/spec.md` § "Exit codes" gains specific entries: `CANCELLED` → 130 (POSIX SIGINT family); `CANCELLED_PLAYLIST_CONFLICT` → 3 (deliberate user-prompt-cancel, distinct from signal-driven); `PARTIAL_FAILURE` → 1; `FAILED` → 1.
- No public API rename or removal. Every change is additive (new typed errors, new optional field on `BIOSResolutionWarning`), narrowing (`Exception` → `(OSError, CopyError)` at `runner.py:259`), or new helper modules.
