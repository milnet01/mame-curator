## What was learned

- **Per-sub-bullet TDD cadence scales to 11 sub-bullets.** Same
  shape as FP20's 12 sub-bullets / 14 commits. Each sub-bullet
  landed in its own commit (sometimes paired, e.g. C+F sharing
  the recyclebin test file), keeping the diff legible.
- **Closing audit on a wide fix-pass surface still finds tests-
  that-don't-lock-their-contract.** Despite per-sub-bullet TDD,
  the closing 4-lane `/indie-review` flagged the FP25-A
  `asyncio.gather` test as a "smoke test masquerading as a race
  test" (no `await` inside the critical section means no real
  race window) and the FP25-F atomicity tests as vacuously true
  (guard branch never taken after FP25-C's rollback). The lesson:
  in TDD, the test passing is necessary but not sufficient —
  ask "does this test fail when the lock is removed?".
- **The "recursive audit returns zero" heuristic (FP10) breaks
  down at sufficient surface area.** FP25 has 11 sub-bullets
  across backend + frontend + tests + docs. The closing `/audit`
  static tools came back clean (the heuristic held there), but
  the 4-lane semantic `/indie-review` found 5 Tier 1. Pattern
  saved: scale closing review depth with surface area.
