## What was learned

- **The cold-eyes loop catches reviewer-hypothesis-bias.** Round 1 surfaced B1 (transitive_missing_in_chain) and B4 (pause/cancel race). Both turned out to be non-bugs after empirical investigation during Step 4. The cold-eyes review can prevent the "user signs off on a partly-fictional plan" failure mode but cannot prevent the "spec authors write things that look like bugs but aren't" failure mode — Step 4 is where reality bites.
- **Recycle path conventions matter.** Cluster R1 surfaced because the FP05 spec wrote "`recycle_root=plan.dest_dir / "data" / "recycle"`" without checking what the rest of the codebase did. The default project-relative `data/recycle/` is the convention; deviating to dest-relative would have orphaned the entries from `purge_recycle()`.
- **Helper extraction surfaces drift.** `read_capped_text` and `atomic_write_text` both came from "two near-identical inline copies that have to track each other" — the threshold where Rule of Three flips. Extracting earlier would have saved the closing-review round but the work was scoped right for FP05 ("first add the helper, then absorb the call sites").
