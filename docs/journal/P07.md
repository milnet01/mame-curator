## What was learned

- **Hardcoding URL defaults is a tradeoff between user-friendly
  out-of-box vs. ship-stale-URL-and-break.** Cluster B chose
  hardcoded AntoPISA-mirror URLs for the 4 mandatory INIs because
  `ManualFallback` already gives the user the URL on failure —
  defaults are convenience, not contract. If a URL goes stale,
  the failure is observable and recoverable. Skipping defaults
  entirely (require user to configure each URL) was rejected
  because most users won't know any URLs.

- **DOMPurify+useMemo is the lower-cost migration path versus
  `react-markdown`.** Both are valid sanitization paths per the
  2026 best-practice (Strapi / HackerOne references). Migrating
  to react-markdown would be cleaner long-term (skips
  dangerouslySetInnerHTML entirely), but it requires rewriting
  the help-content backend rendering pipeline. DOMPurify slot-
  swaps at the existing render seam — closes the same security
  debt with one new import + one useMemo.

- **`useState(initial)` doesn't sync with prop / param changes.**
  HelpRoute's first cut tried `useState(searchParams.get('topic'))`
  to pre-select the topic from the URL — that only fires on mount.
  When the user picks a second topic via Cmd-K (palette navigates
  with a different `?topic=`), the URL changes but selectedSlug
  stays at the first value. Fixed with `useEffect` syncing the
  param into state.

## What was deferred

- **Bundled help markdown content.** P06 shipped the HelpPage
  layout shell; backend `/api/help/*` routes exist (Phase 4 stubs)
  but the actual `docs/help/*.md` content authoring (Quickstart,
  Filters explained, Sessions, Overrides, Playlist conflicts,
  Troubleshooting, Keyboard shortcuts, Glossary) wasn't part of
  cluster E — that's authoring work, not wiring. To be done
  before / during P09's polish pass.

- **`mature.ini` URL.** AntoPISA's GitHub repo doesn't host it.
  P10 (media coverage expansion) or a P12 follow-up could find a
  mirror, but for v1 it stays user-supplied via the existing
  `--mature` CLI flag.

- **Extended help-search ranking.** Cluster E indexes only the
  topic title; body-text indexing for fuzzy "find this concept"
  search wasn't implemented. cmdk's built-in fuzzy matcher works
  on titles, which is enough for the bundled topic count (~10).
