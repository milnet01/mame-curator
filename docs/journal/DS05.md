## What was learned

- **Two-loop cold-eyes review at Step 1 is high-leverage.**
  Loop 1 caught 7 findings (3 HIGH + 3 MED + 1 LOW) that the
  author missed — most load-bearing: the R2 lesson recommendation
  contradicted what `docs/journal/DS02.md` actually asked for
  ("permanent fix" via pre-commit wiring, not "manual checklist").
  Loop 2 caught 9 more (2 HIGH + 4 MED + 3 LOW) — most
  load-bearing: the "Cluster A1 prose conflated the RetroArch
  `it.each` with the Updates banner test" finding, which would
  have produced a wrong-seam split at Step 4.
- **`mcp__ants__cold_eyes_cross_doc_diff` corroboration is useful
  for hotspot detection.** The MCP tool was tested for the first
  time on this project; it identified one corroborated finding
  at `docs/specs/DS05.md:94` across the two reviewer reports
  (different specific concerns, same line). Even with diverse
  specifics, the same-line citation pattern is a real signal —
  reviewers without conversation memory naturally converge on
  the same fragile areas. The tool's regex-based primitive is
  fast and doesn't burn LLM tokens.
- **Pre-commit can run project-specific scripts as `language: system`
  local hooks.** The DS05 D pattern (`pass_filenames: false`,
  `always_run: true`, no `additional_dependencies`) is the
  template for any future CI-only script that needs to mirror
  to pre-commit. The hook runtime is < 1 s; the cost is
  negligible vs the cost of a CI-only-caught drift bug.
- **Leading-underscore module names are the canonical "helper, not
  test" signal in this project.** Both new shared modules
  (`_runner_helpers.py`, `_settingsPageFixtures.tsx`) use the
  pattern. Vitest + pytest both respect it via their default
  test-file patterns (`*.test.ts`, `test_*.py`). The pattern
  scales — future shared test helpers should adopt it.
- **Windows-3.13 lockfile flake in `test_recycle_file_serializes_parallel_sessions`
  re-surfaced during closing CI.** First run failed with
  `PermissionError` on `s1.lock` cleanup; re-run was green.
  This is FP28 A2 territory (the lockfile mechanism's Windows-
  specific edge case the FP28 closing review predicted). Not
  DS05 surface; not actionable from here. Recommended future
  follow-up: a Windows-specific `timeout_ms` cap on the
  lockfile cleanup in `src/mame_curator/copy/recyclebin.py`.
