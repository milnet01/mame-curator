## What was learned

- **Cold-eyes spec review at Step 1 paid off again.** 5 rounds against `docs/specs/P04.md` converged from 23+ findings to clean. Round 1 caught invented Pydantic types (`PickerConfig`, `AppendStrategy`), invented controller method (`is_paused()`), invented `progress_chunk_bytes` runner param, R39 self-contradiction. Without the rounds the implementation would have rebuilt against a partly-fictional contract.
- **Step-3 test-bug surfacing during Step-4 implementation is normal.** Sync `TestClient.get(/api/copy/status)` blocks on SSE; `httpx.ASGITransport` doesn't auto-run lifespan; raw `\n` in URL rejected by httpx pre-server; mini-DAT `listxml` from `tests/copy/fixtures/` lacks pacman/pacmanf cloneof. Each fixed inline with documented comments; not test-quality regressions, just Step-3 author miscalibrations on FastAPI/httpx semantics.
- **Two-pass close-phase is the project's robust shape.** P04 → close attempt → FP09 (13 actionable from indie-review) → close attempt → FP09 Cluster R (5 actionable from FP09's own closing review) → clean → P04 closes. Mirrors the FP01 → FP02 cascade and DS01 → FP05 → FP06 chain. Each fix-pass's closing review on its own patches catches patch-introduced drift; pattern holds.
- **The split lifecycle/progress storage is the natural shape for SSE replay.** The original `Job.history: deque(maxlen=200_000)` looked correct under the C1 test fixture but would have evicted `job_started` on real >200k-event copies. FP09 Cluster R H1 split the storage; lifecycle never evicts, progress is bounded with drop-oldest, replay merges by `ts`. The synthetic-overflow test in `test_fp09_fixes.py` pins it.
