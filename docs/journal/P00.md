## What shipped

- `pyproject.toml` with project metadata, MIT license, Python
  ≥3.12, runtime + dev deps. Phase-staged: runtime deps for
  unbuilt phases live under
  `[project.optional-dependencies].<feature>` until first
  `import` lands in source.
- Ruff configured (line length 100, target Python 3.12, lint
  rules `E F I B UP SIM RUF S D`).
- mypy strict mode (no implicit Optional, disallow untyped
  defs, warn unused ignores).
- pytest with `--cov=mame_curator --cov-report=term-missing
  --strict-markers`; coverage gate at 85%.
- Pre-commit hooks: ruff (check + format), mypy, bandit,
  gitleaks, trailing-whitespace, end-of-file-fixer.
- `.gitignore` + `.gitleaksignore`.
- `LICENSE` (MIT, 2026, Ants Schemel).
- `README.md` skeleton with status + setup section.
- `.github/workflows/ci.yml` matrix (Linux + macOS + Windows;
  Python 3.12 / 3.13).
- `.github/workflows/release.yml` tag-driven, gated on green
  CI per `coding-standards.md` § 12.

## What was learned

- The `[project.optional-dependencies]` pattern (deps move from
  optional → main when the first import lands) keeps fresh-
  install footprint small for early-phase users and stops
  dependency-audit tools complaining about deprecation warnings
  on imports that don't exist yet.
- Astral's Ty was considered but not yet 1.0; mypy strict ships
  for now. Migration when Ty stabilises is a config swap, not a
  code change.
