## What was learned

- **Cold-eyes review caught two scope items the first verification
  pass wrongly dismissed.** Loop 1 of Step 1's cold-eyes review
  re-instated `revision_key_of` memoization and `bytes_by_machine`
  precompute as **Cluster G** — both had been mis-classified as
  "already implemented" by the initial pass when the reality was
  "still per-request work". This is the second time a cold-eyes
  pass on Step 1 has re-instated scope (P15 review found similar)
  — the verification pass needs a second reader.
- **The `schemas/` package directory was a spec mistake.** Spec
  § A5 prescribed `api/schemas/{config,copy,filter,games,_misc}.py`
  but the natural shape turned out to be sibling files under
  `api/` directly (the `schemas.py` facade is still the canonical
  import path; siblings are the implementation detail). Spec
  amended post-hoc in R1c rather than refactoring the working code
  — CLAUDE.md "stay in your lane" applied.
- **Hard-cap exceptions need spec acknowledgement, not silence.**
  A1's `schemas.ts` came in at 591 lines (over the 500 hard cap);
  spec § A1 had silently authorised ≤ 600 without an explicit
  "Deliberately not in scope" entry. R1c adds the entry so the
  next `/audit` doesn't re-raise it.
- **Closing `/audit` returned clean; `/indie-review` caught 3 MED
  + 1 LOW on the DS02 surface itself.** All trivial; all folded
  back inside DS02 as Cluster R1 per the FP02 precedent. The
  audit-vs-indie-review division of labour is holding: audit
  catches tooling-detectable drift, indie-review catches
  contract-vs-implementation drift.
- **lru_cache test isolation is a real concern.** G1's
  `@lru_cache(8192)` is process-wide; without `cache_clear()`
  before stateful assertions in
  `tests/filter/test_picker_revision_memoization.py`, state would
  leak between tests. The test file's two `cache_clear()` calls
  are load-bearing — flagged by the closing review as a
  near-miss.
- **Local gate suite isn't a superset of CI.** The R2 hot-fix
  (post-tag) caught that `tools/check_api_types_sync.py` is wired
  into CI but NOT into the project's local five-gate pipeline or
  pre-commit hooks. Every local invariant DS02 broke was caught
  by the indie-review; this one slipped because the closing
  reviewer didn't think to run a CI-only script. **Closing fix
  in DS05 Cluster D**: `check_api_types_sync.py` wired into
  `.pre-commit-config.yaml` as a local hook
  (`pass_filenames: false, always_run: true`); regression test
  at `tests/tools/test_check_api_types_sync.py` pins the
  PYTHON_SOURCES tuple covers every `api/schemas*.py` sibling at
  HEAD (so a future split fires the test in the same commit).
  Mechanical-drift bugs like A5's "missed 3 sibling modules in
  the discovery tuple" now fail before the close commit, not
  after.
