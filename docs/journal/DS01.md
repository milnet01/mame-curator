## What was learned

- **Cold-eyes spec review at Step 1 catches what plain user review misses.** Round 1 → 4 substantive findings (C9 fully false; C2 read-sites incomplete; A1 third site missed; FP04 needed). Round 4 → 2 (count drift 22→20; FP## allocation deferred). Round 5 → clean. Without these rounds the implementation would have shipped a partly-fictional plan. Saved as `feedback_cold_eyes_spec_review_at_step_1.md` for future projects.
- **The FP02 round-2 precedent generalises.** Every fix-pass's closing review on its own patches surfaces patch-introduced drift, not just leftover round-1 findings. DS01 closing review caught R1 + R2 (both DS01-introduced); pattern holds.
- **C9 was a stale `[Unreleased]` finding.** All flags already had `help=` strings at HEAD — shipped silently in DOC01/P03 but never credited. The lesson: every `[Unreleased]` Tier-3 entry should be re-verified before being folded into a fix-pass; D3 does this for two entries, and it's a cheap general practice for any future debt-sweep.
- **Number-vs-position in ROADMAP.** FP04 was opened pre-emptively (parser deferrals); FP05 emerged from real findings. FP04 < FP05 numerically but FP05 has Tier-1 bugs and runs first in the queue. Numbers reflect creation-order, queue-position reflects priority — both can be true.
