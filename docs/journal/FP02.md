## What shipped

### Tier 2 fixes (3/3)

- **`OverwriteRecord.parent` dropped.** Pre-FP02 the model
  carried `parent`, `old_short`, `new_short`. The runner has
  no `cloneof_map` (FP01 #4 design contract: caller-side
  conflict detection), so it can't compute the actual parent —
  it always wrote `parent=replaced_short=old_short`. Two
  fields holding the same value is misleading. Fixed: dropped
  `parent` from the model; runner instantiation updated;
  rationale captured in the model docstring.
- **`AppendDecision` widened from `StrEnum` to a Pydantic
  model.** New shape: `AppendDecision(kind: AppendDecisionKind,
  replaces: str | None)`. The pre-FP02 enum-only API forced
  the runner into a brittle "first existing-but-not-winner
  basename" heuristic to identify which entry was being
  replaced — and that heuristic returned the wrong entry on
  multi-conflict sessions (e.g. a session replacing both
  `sf2` and `kof94` would always pick `sf2`). With caller-
  supplied `replaces`, the runner trusts the caller's
  identification (consistent with the FP01 #4 design fix that
  pushed conflict detection caller-side). API is breaking but
  pre-v1.0.0; all test sites updated.
- **Recycle dirname keyed on `session_id`.** Was
  `data/recycle/<ISO-timestamp>/` (whole-second resolution),
  which collided across sessions in the same second and the
  second session's `manifest.json` overwrote the first's. New
  format: `data/recycle/<session_id>/`. session_id already
  embeds the timestamp + a 4-byte random suffix, so different
  sessions never collide. Same-session same-name collisions
  (pathological) still walk the existing counter loop.

### Tier 3 fixes (6/6)

- Spec typo `mid-copy3` → `mid-copy` (`copy/spec.md` step 6).
- `_chd_missing(plan)` helper extracted from `run_copy` +
  `_finalize` (was duplicated).
- `functools.partial(on_progress, short)` replaces the
  `make_cb` closure factory for per-file progress callbacks.
- Playlist entries filtered to `SUCCEEDED` +
  `SKIPPED_IDEMPOTENT`. Pre-FP02 the builder iterated
  `(*succeeded, *skipped)` and emitted entries for every
  winner-role outcome — including `SKIPPED_MISSING_SOURCE`
  (source `.zip` missing → `dst` never written) and
  `SKIPPED_EXISTING_VERSION` (KEEP_EXISTING → `dst` never
  written; the existing entry already covers it). Both
  produced `mame.lpl` entries pointing at non-existent files.
  The fix is to require "definitely present at `dst`" — only
  SUCCEEDED (just-written) + SKIPPED_IDEMPOTENT (already at
  `dst` with matching size+mtime). New `copy/spec.md` § "Which
  winners become entries" documents the contract as a
  testable table.
- `test_copy_one_cleans_tmp_on_keyboard_interrupt_with_progress_callback`
  added — covers the chunked-progress branch (`_chunked_copy`)
  that the pre-FP02 KI test missed (it only exercised the
  `shutil.copy2` branch via `progress=None`).
- `test_recycle_three_same_name_same_session_collisions`
  added — three same-name same-session recycle calls walk
  counter=1 → counter=2, hardening the loop bound test.

## What was learned

- **Round-2 indie-review on a fix-pass surfaces fix-pass
  drift, not just leftover round-1 findings.** FP01 round-1
  closed Tier-1; round-2 caught the Tier-2 we shipped here.
  FP02 round-1 closed those; FP02's own closing review
  caught the spec drift the FP02 patches introduced. The
  pattern holds: every fix-pass needs a closing review on
  its own patches, not just on the surrounding code.
- **Spec is the audit surface — drift in the spec is as bad
  as drift in code.** Both reviewers (audit + indie-review)
  flagged the duplicate `AppendDecision` definition before
  the patches even shipped. CLAUDE.md's "no feature merges
  without a spec.md … the test file enforces every clause"
  rule means a self-contradicting spec is a real bug.
- **`functools.partial` over closure factory.** The
  `make_cb` factory was a textbook Python idiom — define a
  factory that returns a closure that captures a name. But
  `partial` is shorter, idiomatic for the "bind the first
  argument" use case, and one less level of indentation.
  Worth applying retroactively where it fits.
- **Caller-side architecture compounds across fix-passes.**
  FP01 #4 pushed conflict detection caller-side; FP02 #2
  pushed conflict-target identification caller-side via
  `AppendDecision.replaces`. The runner is now strictly a
  "trusts the caller" layer — no heuristics, no `cloneof_map`
  reach-around. The CLI / API layer (P04+) becomes the
  natural home for the cloneof-aware decision logic, where
  the data lives.
- **Pre-existing findings stay in their own scope.** The
  audit-triage flagged `runner.py:258` `except Exception
  as exc:` without `logger.exception()` as a Tier-3 finding,
  but correctly noted it was pre-existing (not introduced
  by FP02). Folded into a future debt-sweep, not into FP02
  — keeps the fix-pass scope tight.
