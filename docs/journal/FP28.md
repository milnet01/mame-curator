## What shipped

### Cluster A — Concurrency hardening (3 sub-fixes) `cb35f26`

- **A1** — `JobManager._loop` hoisted from `start()` to `__init__` (with optional `loop` parameter + fallback for sync test fixtures); `_emit` now raises `RuntimeError("JobManager._emit must run on the manager's loop thread")` when called off the loop thread. The check sits after the existing `_current-is-None` early return so FP21-L's no-op-on-cleared-current contract is preserved. `RuntimeError` not `assert` so `python -O` can't strip the guard.
- **A2** — `recycle_file` critical section (counter walk + mkdir + manifest write + `shutil.move`) now wraps in a stdlib `os.O_EXCL` lockfile at `recycle_root / f"{session_id}.lock"`. Serialises parallel sessions in the same `session_id` without pulling a `filelock` dep. Orphan recovery threshold 60 s (~1200x the same-fs p99); same-process contention falls through to a 10 ms retry-sleep.
- **A3** — solved by construction once A2's lock serialises the section: the `target_dir_existed` snapshot can't be invalidated by a sibling session's `mkdir(exist_ok=True)`, so neither rollback path can rmdir a directory another session relies on. Inline comments at the snapshot site and the lock-acquire credit the invariant.

### Cluster B — Correctness regex + extraction + typed errors (5 sub-fixes) `a85307d`

- **B1** — `_LICENSE_RE` developer capture changed from `.+?` to `[^()]+?`. Nested-parens inputs like `"Atari (JSA III) (Williams license)"` now correctly bind publisher=`"Atari (JSA III)"`, developer=`"Williams"` instead of mis-binding to `"Atari"` / `"JSA III) (Williams"`.
- **B2** — `REGION_RE` tightened with a two-branch form: after the region token, allow either (whitespace + comma/close-paren/EOL — `(World)`, `(USA, Set 2)`) or (whitespace + non-uppercase char — `(World 910411)`, `(Europe v2.1)`). Rejects `(World Heroes 2)` because `Heroes` starts with an uppercase H. The lookahead branch was added during cluster-B testing after the initial single-branch form regressed real `(Region YearOrVersion)` patterns enumerated in `tests/filter/test_heuristics.py`.
- **B3** — `Machine.description` switched from `description_elem.text` to `"".join(description_elem.itertext()).strip()`. Mixed-content `<description>Foo <i>bar</i> baz</description>` now yields `"Foo bar baz"` rather than truncating at `"Foo "`. Defensive — MAME DATs don't currently ship mixed-content in `<description>`.
- **B4** — `filter.runner` declared a module logger that no path called. Three override-rejection sites (parent not in groups; target not in DAT; target wrong group) now call `logger.warning(msg)` alongside the existing `FilterResult.warnings.append(msg)`. Dual-channel contract per `filter/spec.md` § Phase C.
- **B5** — `_apply_session` subscript `sessions.sessions[sessions.active]` wrapped in `try / except KeyError`; re-raises as `SessionsError(FilterError)`. Reachable only via Pydantic v2 `model_copy` (which skips validators); direct construction is blocked by the existing `model_validator`.

### Cluster C — Boundary hardening (2 sub-fixes) `0281695`

- **C1** — `_validate_paths` now gates `paths.retroarch` (POSIX `os.access(p, os.X_OK)` / Windows `shutil.which` for extension-based exec check) and `paths.retroarch_core` (`.exists()` on both platforms — cores are `dlopen`/`LoadLibrary`'d, not directly executable). Closes the PATCH-config → launch chain; pre-fix a malicious PATCH could land `paths.retroarch=/usr/bin/evil-thing` which `api/routes/games.py:275` would hand to `subprocess.run`.
- **C2** — `media_proxy`'s hardcoded `media_type="image/png"` replaced with `mimetypes.guess_type(str(path))[0] or "image/png"` (suffix-sniffed); added `Cache-Control: public, max-age=2592000, immutable` per design § 6.3. Pre-fix every page-load re-fetched libretro thumbnails despite the permanent on-disk cache.

### Cluster D — CLI exit-code + error-surface drift (3 sub-fixes) `8fe7641`

- **D1** — `serve` now wraps `uvicorn.run` in `try / except KeyboardInterrupt: return 130` (defence-in-depth — uvicorn currently catches Ctrl-C internally) and the trailing return changed from 0 to 130 so the function honours POSIX convention regardless of which side ends up catching the signal.
- **D2** — bare `except Exception` around `create_app()` narrowed to `(ConfigError, ParserError, FilterError)`. Programmer errors (RuntimeError, AttributeError, ...) now propagate as tracebacks instead of being squashed into a one-line stderr message. (R1.2 noted the catch is currently dead at the immediate call site because `create_app` is a pure factory — kept as defence-in-depth for a future validation refactor; comment amended to name the constraint.)
- **D3** — three inline imports in `_cmd_refresh_inis` (asyncio, httpx, mame_curator.updates) lifted into a `try / except ImportError` mirroring `_cmd_serve`'s guard. Defence-in-depth pattern consistency for exotic install states (pip install --no-deps, broken wheel, partial editable install).

### Cluster E — Design § 6.7 deferral (1 ADR + 1 cross-link) `8dcae9d`

- **E1** — new ADR at `docs/decisions/0004-ini-refresh-trust-model.md` recording the wizard-vs-refresh split (design § 6.6 promised mirrors + sha256 for the wizard bootstrap; § 6.7 runtime refresh stays silent on integrity), current refresh trust posture (HTTPS-only, AntoPISA repo integrity, no per-file sha256), and the post-v1 hardening path. ### Cluster R1 — Closing-review fold-in (3 corrections) `72505d8`

Closing `/indie-review` on the `cb35f26..8dcae9d` surface surfaced one HIGH and two MEDIUM findings on the changeset itself. All folded:

- **R1.1** (HIGH) — C2 tests were dead. The original draft had `if response.status_code != 200: return` which short-circuited the cache-control assertion without raising when the upstream wasn't mocked; the sniff test was a bare `pytest.xfail` with no body. - **R1.3** (MEDIUM) — FP28.md § B2 spec text described a single-branch regex tightening, but the actual shipped implementation needed a two-branch form. Updated the spec body to document the two-branch lookahead form that shipped, naming the `(World 910411)` regression that drove the refinement.

## What was learned

- **Cold-eyes-loop verification catches test-construction bugs the implementer can't.** Lane 4 loop 2 caught two critical test-construction errors in the spec's `Tests to write first` section before they shipped (Test 8 wrong FieldError shape; Test 12 over-engineered monkeypatch). Without the lane the bugs would have landed in the source-test commits and surfaced as red CI runs.
- **`pytest-fast` pre-commit hook precludes RED-state tests-only commits.** Carried over from FP27 / DS04 — the project ships **tests + implementation together per cluster commit**, not a "tests first, impl second" two-commit pattern. Step 3 of the 9-step App-Build loop adapts to this: tests are drafted on disk during Step 3 but commit only when the matching cluster's impl lands in Step 4. The original FP28 spec drafted a "batch 0 RED tests commit" framing that had to be corrected mid-flight (commit `c1b6888`) once the hook blocked it.
- **`git stash --keep-index` doesn't include untracked files.** The pre-commit framework stashes unstaged tracked-modified files but not untracked ones, which means new test files from later clusters stayed on disk during earlier per-cluster commits and broke the hook's pytest run. Worked around by moving untracked files to `/tmp/fp28-pending/` before per-cluster commits, restoring them per cluster. A cleaner pattern for the next multi-cluster fix-pass: stage everything once, commit per-cluster via `git commit -- <files>` (the staging area protects the working tree from disruption). Worth a small skill-update for `/close-phase` or the App-Build workflow.
- **Spec text drifts during implementation when the spec describes a simpler form than reality requires.** B2's spec said "tighten with `\s*(?:,|\)|$)`" — but that broke `(World 910411)` because real region parentheticals routinely contain year/version continuations. The implementation grew a second branch with a lookahead; the spec was retroactively updated in R1.3. The pattern: when implementation diverges from spec, fix the spec at the same SHA — don't ship divergence.
- **C2's content-type sniff is currently dead in production.** `cache_path_for` derives the on-disk extension from `urlparse(url).path` suffix, and all current libretro URLs end in `.png`. The sniff fix only kicks in when a JPG-URL source is added in a future phase — the test pins the contract via `mimetypes.guess_type` directly but doesn't exercise a JPG cache file end-to-end. Flagged in test docstring; a follow-up should mock a JPG-URL source when one lands.

## What was deferred

Two items flagged out-of-FP28 during R1 triage:

- **C2 sibling** — `media_proxy`'s four error branches lack negative-cache headers; browsers may cache 502/404 per HTTP defaults, silently masking transient upstream failures. Pre-fix had no header at all so this is not a regression; flag for DS02 or a dedicated media-cache hardening pass.
- **A2 lockfile timeout cap** — the retry loop has no bounded-attempt cap. The 60 s stale-reclaim threshold is the only failsafe; a holder that's alive but slow (cross-FS multi-GB move per design § 6.4) would have the waiter spin indefinitely. Defensive cap can land in a later hardening pass.

Plus two pre-existing follow-ups carried over:

- **Missing `updates/spec.md`** — flagged in FP28's E1 scope as "the only shipped module without a per-module spec". Out-of-FP28 because authoring a per-module spec for an existing module is itself a spec-grade task (cold-eyes loop, etc.). Should be the trigger for a dedicated DOC## or a P12-class pass when the wizard work lands.
- **Tier 3 structural debt** — DS02 already queued as the next item after FP28 closes per workflow.md § 1.
