# P01 — DAT + INI parsers (`parser/`)

- **Status:** ✅ closed 2026-04-27
- **Original phase label:** Phase 1
- **Long-form contract:**
  [`docs/superpowers/specs/2026-04-27-roadmap.md` § Phase 1](../superpowers/specs/2026-04-27-roadmap.md)
- **Spec:** [`src/mame_curator/parser/spec.md`](../../src/mame_curator/parser/spec.md)
- **Theme:** pure-Python parser turning the user's DAT XML and the
  five `.ini` reference files into typed in-memory data; CLI smoke
  prints summary stats.

## What shipped

- `Machine` — frozen Pydantic model with `extra="forbid"`.
- `parse_dat(path)` — `lxml.iterparse` streaming; handles `.xml`
  and `.zip` (auto-extract single XML); 43k machines on the user's
  real DAT in <30s.
- `parse_catver(path)`, `parse_languages(path)`,
  `parse_bestgames(path)`, `parse_mature(path)`,
  `parse_series(path)` — all over a shared `_parse_simple_ini()`
  helper.
- `parse_listxml_disks(path)` — returns `set[str]` of machine
  names with `<disk>` entries (CHD-required).
- `split_manufacturer(s)` — splits `"Capcom (Sega license)"` into
  `(publisher="Capcom", developer="Sega")`.
- `mame-curator parse <path>` CLI subcommand printing total
  machines, parents, clones, bios/devices/mechanical counts,
  CHD-required count.
- Coverage on `parser/` ≥ 90%.

## What was learned

- **Pleasuredome DATs strip `cloneof` and `romof`.** Verified
  directly: `grep -c cloneof= MAME\ 0.284\ ROMs\ \(non-merged\).xml`
  returns `0`. Recorded in P01 spec "Edge cases" and surfaced in
  the P02 design as the cloneof-from-listxml decision (now
  ADR-0002).
- `lxml.iterparse` + `.clear()` per machine keeps memory flat at
  ~30 MB even on the 48 MB / 43k-entry DAT. `etree.parse` would
  spike to ~250 MB during parse.
- `manufacturer` field in the DAT carries publisher + developer
  in one string. The "(license)", "(licensed from)", "(licensed
  by)" patterns all map cleanly; the user's real DAT had no edge
  case the heuristic missed.
