## What was learned

- **Closing-review-of-closing-review pays off again.** The FP02 / DS01 / FP05 / FP06 / FP08 precedent held: the closing `/indie-review` on FP09's own patches surfaced 4 findings that the initial implementation missed. H1 in particular was the highest-ROI catch — the original `_HISTORY_CAP = 200_000` would have shipped looking correct, with a regression test that only exercised tiny fixtures, while a real-world 10k-file copy would silently break SSE replay.
- **False-positive triage was correct on three.** B1 (Session round-trip), B2 (parser-level listxml combine), B8 (`_help_dir()` arithmetic) were all dismissed during FP09 implementation; the closing review re-verified each — Session validator forces non-default include rules; parser changes violate anti-jump rule 2; the path arithmetic is correct (reviewer miscount). All three rationales documented.
- **`heapq.merge` over two ordered streams is the right shape.** Both lifecycle list and progress deque are appended in time order from a single emitter. Streaming merge is O(N log K) with K=2; on a 230k-event replay (30k lifecycle + 200k progress) that's a one-time cost on subscriber connect, well below SSE consumer's tolerance.
- **Spec-anchor discipline matters even for cosmetic-looking findings.** Open Q2 (250 ms vs 50 ms drift) was a one-character textual mismatch but flagged a real spec/code divergence. Per coding-standards § 0.1 ("never silently drift"), the rule that catches it is "every fix-pass closing review re-reads the spec lines the patches touched."

## False positives confirmed

- **B1** — Session round-trip via `exclude_defaults=True`. **Why FP**: `Session._validate_session` at `filter/sessions.py:34` raises `ValueError("session has no include rules")` on any session whose four `include_*` fields are all default-empty. So no valid Session has all-default fields; round-trip is lossless on the universe of valid inputs.
- **B2** — Triple `parse_listxml_*` on startup. **Why FP**: combining requires a `parser/listxml.py` change to expose a unified iterparse callback API. That violates anti-jump rule 2 ("no enlargement of earlier modules"). Real-listxml startup cost (~5–10s on a 370 MB file) is acceptable for a server lifespan.
- **B8** — `_help_dir()` arithmetic. **Why FP**: reviewer miscount. From `src/mame_curator/api/routes/help.py`, `parents[0]=routes`, `parents[1]=api`, `parents[2]=mame_curator`, `parents[3]=src`; `parents[3].parent / "docs" / "help"` correctly resolves to `<repo_root>/docs/help`. Test `test_b8_help_dir_arithmetic_lands_at_repo_docs_help` confirms.

No allowlist entries added (these are reviewer-side false positives, not tool false positives — different surface).
