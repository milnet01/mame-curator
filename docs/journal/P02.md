# P02 — Filter rule chain (`filter/`)

- **Status:** ✅ closed 2026-04-27
- **Original phase label:** Phase 2
- **Long-form contract:**
  [`docs/superpowers/specs/2026-04-27-roadmap.md` § Phase 2](../superpowers/specs/2026-04-27-roadmap.md)
- **Spec:** [`src/mame_curator/filter/spec.md`](../../src/mame_curator/filter/spec.md)
- **Theme:** deterministic curated set from parsed data — drop +
  pick + override + session-slice rule chain, with a JSON report
  CLI subcommand.

## What shipped

- `parse_listxml_cloneof(path)` — extends `parser/listxml.py`;
  returns `{clone_short_name: parent_short_name}` (per ADR-0002).
- `region_of(description)` and `revision_key_of(description)` —
  pure heuristic functions in `filter/heuristics.py`.
- **Phase A drops** — small predicates per rule (BIOS, device,
  mechanical, category prefix, language-Japanese-only, preliminary
  driver, CHD-required, genre allow/deny, publisher/developer
  allow/deny, year-before/after).
- **Phase B picks** — comparators composed via
  `functools.cmp_to_key`; preferred-genre / -publisher /
  -developer boost. (Pass-3 indie-review fix: was using a
  score-tuple + `max()` that broke on alpha fallback;
  `cmp_to_key` per spec line 55 is the contract.)
- **Phase C overrides** — `apply_overrides(decisions,
  overrides_yaml) -> decisions`. Unknown override warns but
  doesn't crash.
- **Phase D session focus** — active session slices the visible
  set; null session returns full library.
- `run_filter(parsed, config, overrides, cloneof_map) ->
  FilterResult` orchestrator. Frozen Pydantic `FilterResult`
  with `winners`, `dropped`, `contested_groups`.
- `mame-curator filter` CLI subcommand outputting a JSON report.
  Dispatch via `set_defaults(func=...)` (pass-3 indie-review fix).
- 158 tests pass. Coverage on `filter/` ≥ 96%. Snapshot test on
  hand-picked 30-machine fixture; hypothesis property tests for
  determinism + idempotency.
- CLI smoke against user's real data: deterministic across runs
  (verified by running twice and diffing the report).

## What was learned

- **Pass-3 indie-review surfaced two CRITICAL spec violations and
  one HIGH zombie field.** All three closed before P03 advanced:
  - **Picker not using `cmp_to_key`.** The score-tuple approach
    appeared correct on most inputs but broke on prefix-collision
    alpha fallback (`sf2ce` was winning over `sf2`). Pin
    test: `test_alphabetical_fallback_spec_lower_wins_with_prefix_collision`.
  - **CLI dispatch not using `set_defaults(func=...)`.** Migrated;
    `cli/spec.md` updated to make the migration the current
    contract.
  - **`drop_bios_devices_mechanical` config field declared but
    never honoured** — zombie field. Wired into Phase A
    predicates 1-3 with early-return when `False`. `filter/spec.md`
    Phase A table updated.
- Tier 2 / Tier 3 findings logged in CHANGELOG `[Unreleased]` per
  the project's CHANGELOG-as-sweep-log convention.
- The triage discipline (pass-by-pass, with each tier explicitly
  ranked) caught issues a single-pass review would have missed.
