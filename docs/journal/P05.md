## What changed

3 source files in `src/mame_curator/media/` (~120 LoC) implementing the libretro-thumbnails URL builder + sha256-keyed lazy-fetch disk cache per `docs/specs/P05.md`:

- `media/urls.py` — `escape_libretro(name)` applies the 10-character filename rule (`&*/:\<>?|"` → `_`); `urls_for(machine: Machine) -> MediaUrls` runs escape → `urllib.parse.quote` → concat into the libretro-thumbnails URL space (`Named_Boxarts/`, `Named_Titles/`, `Named_Snaps/` under `https://raw.githubusercontent.com/libretro-thumbnails/MAME/master`). `MediaUrls` is a frozen Pydantic model with `extra="forbid"`; **the absence of a `video` field is load-bearing** — the R39 short-circuit on `kind == "video"` exists because `getattr(MediaUrls, "video")` raises `AttributeError`. Design §6.3 routes video through progettoSnaps (P06+), not libretro-thumbnails.
- `media/cache.py` — `cache_path_for(url, cache_dir) -> Path` is a pure helper (`sha256(url).hexdigest()` + URL-suffix-derived extension); no I/O, no directory creation. `async fetch_with_cache(url, cache_dir, *, client) -> Path | None` returns the on-disk path on cache hit, downloads via the caller-supplied `httpx.AsyncClient` on miss, atomic-writes via `_atomic.atomic_write_bytes`, returns `None` on upstream 404 (no negative caching), raises `MediaFetchError` on other upstream/network failures or empty-body `200`s. Typed exceptions: `MediaError` base; `MediaFetchError` for upstream/network. The `path.exists()` race is benign because cache writes are append-only via `atomic_write_bytes` (no delete path) — invariant comment makes the contract visible.
- `media/__init__.py` — re-exports the public surface: `MediaError`, `MediaFetchError`, `MediaUrls`, `cache_path_for`, `escape_libretro`, `fetch_with_cache`, `urls_for`.

R39 (`/media/{name}/{kind}` in `api/routes/media.py`) swapped from inline URL build + bare `client.get` to `urls_for` + `fetch_with_cache`. The handler stays under 30 lines; `kind=video` short-circuits with `MediaUpstreamNotFoundError(404)` BEFORE calling `urls_for` (load-bearing). `app.state.media_client` migrated from bare `httpx.AsyncClient()` to `httpx.AsyncClient(timeout=10.0, follow_redirects=True)` in lifespan setup — 10s timeout moves from per-call to client construction; `follow_redirects=True` ensures libretro CDN 301/302 transits transparently (FP10 A1).

47 net new tests across 3 files in `tests/media/` (23 escape + 6 urls + 12 cache) + 6 integration tests in `tests/api/test_routes_media.py` + `media_cache_dir` fixture in `tests/api/conftest.py` for tmp_path-isolated per-test cache (avoids CWD-relative `./data/media-cache` cross-pollution between tests). Plus the existing 376 tests project-wide → **423 total pass on close. Coverage 89.12%; `media/cache.py` 100%, `media/urls.py` 100%, `media/__init__.py` 100% (above 90% gate).**

## What was learned

- **Cold-eyes spec review at Step 1 paid off again.** Two rounds against `docs/specs/P05.md` converged from 9 round-1 findings (3 Tier-1 + 5 Tier-2 + 4 Tier-3) → 2 round-2 findings (1 Tier-2 + 1 Tier-3) → clean. Round 1 caught the 10-character escape rule (the design doc + roadmap rendering broke across an unbalanced backtick — the canonical character set was implicit; the spec now pins it explicitly), the load-bearing absence of `MediaUrls.video`, the `httpx.AsyncClient(timeout=10.0)` implementation prerequisite, the `cache_path_for` purity contract, and the concurrency note (atomic-write makes torn writes impossible at single-user scale). Without the rounds the implementation would have re-derived the escape rule from the malformed markdown, missed the video short-circuit's structural origin, and shipped a per-call timeout footgun.
- **Two-pass close-phase remained the project's robust shape.** P05 → close attempt → FP10 (5 actionable from indie-review) → clean → P05 closes. Mirrors the P03→FP01→FP02 cascade, DS01→FP05→FP06 chain, and P04→FP09 close. FP10 was the fastest fix-pass converge yet (one round, no Cluster-R) — the small surface (113-line patch across 6 files) didn't generate patch-introduced drift. Pattern: smaller fix-passes converge faster.
- **`/indie-review --fix HEAD` was the right shape for FP10's narrow surface.** Full-sweep `/indie-review` is calibrated for ≥5 commits across ≥3 subsystems; FP10's 5 findings on 4 files don't earn that cost. Dispatching one focused subagent against the FP10 commit diff with the ROADMAP entry as the contract took ~60s vs the multi-lane sweep's ~5 min, and produced tighter signal because the agent's mandate was "judge whether each numbered finding is closed" rather than "scan everything for blind spots." Adopting as the standard fix-pass close shape.
- **R39's `kind=video` short-circuit is load-bearing structurally, not just behaviorally.** A naive reading would say "video returns 404 because we haven't wired progettoSnaps yet" — easy to refactor away by adding `video` to `MediaUrls` and pointing it at progettoSnaps. The actual contract is stronger: `MediaUrls` has no `video` field (the type-system encodes the deferral), so `getattr(urls, kind)` for `kind == "video"` would raise `AttributeError` if the short-circuit weren't there. The spec calls this out at line 60 ("Load-bearing consequence"); the route's comment at `routes/media.py:42` echoes it. P06+ wiring of progettoSnaps will need to add the field AND remove the short-circuit AND extend the test that asserts `getattr(MediaUrls(...), 'video')` raises `AttributeError` — three coordinated changes, not one.
- **Test isolation via `media_cache_dir` fixture caught a real cross-test bug during Step-4.** Initial impl had two `tests/api/test_routes_media.py` tests share the default `./data/media-cache/` path; the second test in run order found the cache pre-populated by the first and never hit upstream, hiding the bug the test intended to assert. tmp_path-per-test fixture isolation is the right pattern; landed both as the fixture in `conftest.py` and as a one-line note in the spec under "R39 wiring" so future test authors don't re-trip the trap.

## What was deferred

- **Mass pre-fetching / batch warmup.** Always lazy by design — the frontend's `<img src>` populates the cache organically as the user browses (P06+).
- **Cache eviction / size cap.** Permanent by design (§6.3); a Settings-page "Clear cache" button is P06+ when the Settings UI lights up.
- **Video thumbnails via progettoSnaps.** Design §6.3 routes video through progettoSnaps; that's P06+ when the Settings page wires the opt-in. P05's R39 returns `media_upstream_not_found` for `kind=video` with no upstream round-trip.
- **Per-URL fetch deduplication via `asyncio.Lock`.** Wasted second downloads on concurrent same-URL requests are acceptable at single-user scale — `atomic_write_bytes` prevents torn writes regardless. The lock would couple `media/` to the event loop in a way that doesn't pay off until multi-user scale (post-v1).
- **Content-type sniffing.** Trust the URL suffix; libretro-thumbnails serves `.png` exclusively. A future progettoSnaps wiring (P06+) for `.mp4` video will need a `mediaType` parameter; deferred until that wiring is real.
