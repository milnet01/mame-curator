**Notable post-close follow-up.** The A2 `_SPAStaticFiles` carve-out shipped with a Linux-tested `path.startswith(("api/", "media/", "assets/"))` check that worked on Linux + macOS but missed on Windows because Starlette's `StaticFiles.get_response` joins URL paths against the on-disk dir using `os.path` — Windows produces backslash-separated paths (`api\typo` instead of `api/typo`), so the prefix check failed and the SPA fallback cascaded for typo'd API paths and missing assets. ## What was learned

- **CI's Windows + Linux + macOS matrix is load-bearing for code that touches paths.** The A2 carve-out passed local Linux pytest, the project's Linux dev box, and a manual macOS CI run — only failed on Windows. Without the Windows job, the regression would have shipped silently. Saved as a reminder: any code that touches path strings under `StaticFiles` / `os.path` machinery needs explicit Windows coverage, and the cheapest way to get it is a property test against the underlying constants rather than a full TestClient round-trip.
- **Closing audits scale with patch surface.** FP11's patch range is ~25 commits across ~80 files, vs FP10's 113-line patch across 6 files. The full-sweep `/indie-review` returned ~40 findings; FP10's `/indie-review --fix HEAD` returned zero. The shape of the closing review should match the shape of the surface — narrow fix-passes get `--fix HEAD`, wide phase-closing fix-passes get the multi-lane sweep.
- **Container components are the right shape for hook-owning route elements.** B8's pattern — `SessionsRoute` / `ActivityRoute` / `StatsRoute` / `HelpRoute` as 5-15-line inline functions in `App.tsx` that read their hook(s) and thread results into the existing page prop contract — kept the page components pure-prop / pure-render (no hook calls inside the page) so the existing component tests still passed unchanged. Pulling containers into a new directory would have been ceremony; inlining beat it.
- **Drift-gate hardening compounds.** Cluster C tightened the `check_api_types_sync.py` brace-balanced TS parser, scoped `_inherits_basemodel` to the real Pydantic class, and made duplicate-class detection fail loudly. Each fix was small (one regex / one isinstance check / one raise) but the combined gate now refuses to silently mis-sync — every actual drift surfaces at the gate.
- **Mount-on-key beats setState-in-effect for "reset state when prop changes".** D1's `NotesEditor` rewrite went through three intermediate forms — derived state via ref comparison, `useEffect(initial → setDraft)`, then mount-on-key. The mount-on-key contract (parent passes stable `key={gameShortName}`) is the only form that survives the React 19 compiler's "no setState in effects" lint AND avoids the stale-`Saved` flash race when navigating between games. The trade is documented in `NotesEditor.tsx`'s docstring; future React-19 components facing the same "reset state on prop change" decision should reach for mount-on-key first.

The B8 container hook wiring covers data flow; the actual UI primitives for editing list-shaped config (region_priority, include_genres, allowed_roots) and the `<FsBrowser>` path picker are FP12's domain. P07 was amended to mention `cards_per_row_hint` UI control as part of the Settings expansion since FP12 introduces the primitives that enable it.
- **`/audit` + `/indie-review` (FP11 closing pass).** The user opted to ship FP11 + P06 on CI green (multi-platform matrix proves the patches don't introduce platform-specific regressions) rather than dispatch the multi-agent closing audit a second time. The pattern saved is "user-elected close-on-CI-green is acceptable for fix-passes whose patches are themselves the result of an indie-review pass" — running indie-review on the patches that closed indie-review findings is the kind of recursive audit that FP10's "smaller fix-passes converge faster" lesson predicts will return zero findings. The FP10 closing-review-of-closing-review precedent still holds for narrower fix-passes; for FP11's wide surface the user's call was to trust CI + the per-cluster commit gates.

## False positives confirmed

- **`allowlist-002`** (`useVirtualizer` library-by-design warning). `@tanstack/react-virtual`'s `useVirtualizer` returns a virtualizer instance whose `.getTotalSize()` and `.getVirtualItems()` are stable across renders by library design; the React Hooks lint warns that they "may change" and demands them in the dep array of the `useMemo` they're consumed in. Adding them is incorrect (they're stable) and removing the lint exemption would force a rewrite away from the official library API. Allowlisted at the per-file level in `eslint.config.js`.
- **`allowlist-003`** (shadcn-vendored Fast-Refresh warnings on `components/ui/**`). The shadcn CLI vendors primitives that mix component exports + utility exports (`badgeVariants` + `Badge` in the same file). The Fast-Refresh lint warns about non-component exports in component files; rewriting the vendored primitives would break the upgrade path (`npx shadcn add` would re-introduce the warnings on every regeneration). Allowlisted via `eslint.config.js` overrides for `components/ui/**`.
