# P03 — Copy + BIOS resolution + RetroArch playlist

- **Status:** ✅ closed 2026-04-30 (paired with FP01 close)
- **Original phase label:** Phase 3
- **Long-form contract:**
  [`docs/superpowers/specs/2026-04-27-roadmap.md` § Phase 3](../superpowers/specs/2026-04-27-roadmap.md)
- **Spec:** [`src/mame_curator/copy/spec.md`](../../src/mame_curator/copy/spec.md)
- **Theme:** given a Phase-2 winner list, copy each winner's `.zip`
  plus the transitive BIOS chain to the destination, write a
  RetroArch `mame.lpl` playlist, emit a copy report, and append
  one or more events to `data/activity.jsonl`. CLI:
  `mame-curator copy --dry-run` / `--apply` / `--purge-recycle`.

## What shipped

- **`parser/listxml.py` extension:** `parse_listxml_bios_chain(path) -> dict[str, BIOSChainEntry]`
  capturing `romof` + `<biosset>` per machine. Same lxml fast-iter
  pattern as `parse_listxml_disks` / `parse_listxml_cloneof`.
- **`copy/bios.py`:** `resolve_bios_dependencies(winners, chain)`
  walks the chain transitively with cycle safety (seen-set +
  `entry.romof != name` guard). Missing-from-listxml winners
  emit a non-fatal `BIOSResolutionWarning(kind="missing_from_listxml")`.
- **`copy/executor.py`:** `copy_one(src, dst, *, short_name, role, progress=None) -> CopyOutcome`
  with `.tmp` + `os.replace` atomicity, mtime preserved via
  `shutil.copystat` / `shutil.copy2`, idempotent on size+mtime
  match, chunked progress callback. **`try/finally` ensures
  `.tmp` cleanup on every exception path including
  `KeyboardInterrupt`** (FP01 fix).
- **`copy/playlist.py`:** `write_lpl()` produces RetroArch v6+
  JSON with canonical key order, UTF-8 no BOM, ensure_ascii=False,
  atomic via `.tmp` + `os.replace`. `read_lpl()` parses v1.5+
  JSON only (legacy 6-line format deferred per FP01).
- **`copy/preflight.py`:** missing-source / dest-writable /
  existing-playlist / idempotent-already-copied detection.
- **`copy/recyclebin.py`:** project-internal `data/recycle/`
  with 30-day retention purge; per-batch `manifest.json`. Clean
  counter-loop on filename collisions (FP01 fix replaced a
  broken `for _ in [None]` 1-shot generator).
- **`copy/activity.py`:** append-only `data/activity.jsonl`;
  reverse-line reader skips corrupt lines.
- **`copy/controller.py`:** `CopyController` state machine with
  `threading.Event`-driven pause/resume; sticky cancel; pause
  holds at file boundary.
- **`copy/runner.py`:** `run_copy(plan, controller=None, on_progress=None) -> CopyReport`
  orchestrator. CANCEL strategy bypassed on idempotent rerun
  (design § 6.4 idempotency contract). REPLACE /
  REPLACE_AND_RECYCLE branches populate `OverwriteRecord` and
  recycle the displaced zip (FP01 fix). Corrupt-or-legacy
  existing-playlist surfaces a warning instead of silent
  overwrite (FP01 round-2 fix — user-data risk closed).
- **CLI** `mame-curator copy --dry-run / --apply / --purge-recycle`
  with `--conflict {append,overwrite,cancel}` and
  `--delete-existing-zips`.
- **74 tests** in `tests/copy/` covering BIOS chain, executor
  atomicity + idempotency + KeyboardInterrupt cleanup, playlist
  format + roundtrip + Unicode + error branches, preflight,
  recycle (retention, same-name collision), activity log
  (append + corrupt-line tolerance + discriminator round-trip),
  controller (pause/resume/cancel state + thread safety), runner
  (golden path, missing source, APPEND/OVERWRITE/CANCEL,
  REPLACE + REPLACE_AND_RECYCLE OverwriteRecord population,
  corrupt-playlist warning, FAILED branch, OVERWRITE+delete
  recycling, pause-holds, cancel-keeps-partial), CLI smoke.
- **Coverage:** copy/ aggregate ~89% (gate ≥85%); 92.95% project-
  wide. All five CI gates green.

## What was learned

- **Two reviewers, two verdicts.** `/audit` (audit-triage:
  ruff/mypy/bandit/pytest-cov/grep) returned 0 Tier-1 — clean
  pass. `/indie-review` independently flagged 6 Tier-1 real bugs
  that automated tools missed: signature drift, missing
  `KeyboardInterrupt` cleanup, `OverwriteRecord` allocated-but-
  never-appended, broken recycle collision logic, spec/code
  drift on `read_lpl` legacy support, and the append-decision
  contract being unenforceable from the runner layer (which
  doesn't have `cloneof_map`). The lesson: automated audits
  catch the *grammar* of code; indie-review catches the
  *semantics*. App-Build's "every finding tracked" rule kept
  these from slipping into production.
- **Spec vs code: when they disagree, sometimes the spec is
  the bug.** The original spec mandated `PlaylistError` on
  missing append-decision for "a conflicting winner" — but the
  runner can't detect "conflicting" without `cloneof_map` (which
  lives one layer up in `FilterContext`). Resolution: update the
  spec to make conflict detection caller-side (CLI / API uses
  its `cloneof_map`; runner trusts presence-in-`append_decisions`
  as the conflict signal). Cleaner architecture, less coupling.
- **`try/finally` over `try/except` for cleanup paths.**
  Catching only `OSError` is the easy default but misses
  `KeyboardInterrupt`, `MemoryError`, and any unexpected
  exception from a callback. The cleanup belongs in `finally`.
- **Project-internal recycle bin is the right call.** No
  `send2trash` dependency, no cross-platform wheel, full control
  over the 30-day retention contract. The user owns the data;
  the project owns the audit trail.
- **Round-2 indie-review surfaced fresh-eyes findings** even on
  patched code: 5 Tier-2 + 4 Tier-3 not seen in round-1.
  Resolution: fixed the user-data-risk one (B-T2-3) inside FP01;
  deferred the rest to FP02 to keep the close from sprawling.
  Per App-Build, "every finding tracked" — they're all in the
  ROADMAP.
