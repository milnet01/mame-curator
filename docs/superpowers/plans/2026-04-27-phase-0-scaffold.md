# Phase 0 — Scaffold and Tooling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a bare `mame-curator` repository where all tooling — uv, ruff, ty, pytest, pre-commit, GitHub Actions CI — runs successfully on an empty project. No business logic yet; just the scaffold.

**Architecture:** Modern Python project using uv as the package/env manager, src/ layout (the de-facto 2026 standard), Astral's Ruff + Ty for lint and type checking, pytest for tests, pre-commit hooks mirroring CI, GitHub Actions matrix on Linux/macOS/Windows × Python 3.12/3.13. The frontend scaffold is intentionally deferred to Phase 6.

**Tech Stack:** Python ≥ 3.12, uv (Astral), Ruff (Astral) ≥ 0.9, Ty (Astral; mypy fallback), pytest ≥ 8, pytest-cov, pytest-asyncio, hypothesis, respx, bandit, gitleaks, pre-commit ≥ 4, GitHub Actions.

- [Coding standards](../../standards/coding-standards.md)

```markdown
# MAME Curator

> Curate the best playable arcade games from a MAME ROM set, with a modern web UI.

**Status:** pre-alpha. Active development on `main`. APIs and config formats may change without notice until v1.0.0.

## What it does

- Reads a MAME non-merged ROM set + DAT XML.
- Drops non-arcade entries (BIOSes, devices, computers, console BIOSes, mahjong/casino/etc.) using community reference data (`catver.ini`, `languages.ini`, `bestgames.ini`, ...).
- Picks the best version from each parent/clone group via a deterministic rule chain (community ratings → parent over clone → driver status → region → revision → ...).
- Lets you override picks through a modern web UI with cover art, screenshots, and side-by-side comparisons.
- Copies the chosen ROMs (plus required BIOSes) to a separate destination, leaving the source untouched.
- Generates a RetroArch playlist (`mame.lpl`) so games show pretty descriptions in RetroArch without renaming files.

See the [design spec](docs/superpowers/specs/2026-04-27-mame-curator-design.md) for the full picture.

## Requirements

- Python ≥ 3.12
- `uv` ≥ 0.5 ([install](https://docs.astral.sh/uv/getting-started/installation/))
- A modern web browser
- A MAME non-merged ROM set + matching DAT XML (you supply these)

## Task 11b: Release workflow with CI-must-pass gate

**Files:**
- Create: `.github/workflows/release.yml`
- Modify: `CHANGELOG.md` — note the release workflow

The release workflow is triggered by a tag push matching `v*.*.*`. It runs the full CI gate against the tagged commit; only if every check passes does the publish job create a GitHub Release with the build artifact attached. **CI failure → no release.** This is the enforcement of the standards §12 rule.

- [ ] **Step 1: Write `.github/workflows/release.yml`**

```yaml
name: Release

on:
  push:
    tags:
      - "v*.*.*"

# Release runs are sequential per ref; never cancel a release in flight.
concurrency:
  group: release-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: write   # required to create the GitHub Release

jobs:
  # Stage 1 — re-run the full CI gate at the tagged commit.
  # If any of these fail, the publish job never runs and no Release is created.
  verify-ci:
    name: Verify CI is green at tag
    strategy:
      fail-fast: true
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        python-version: ["3.12", "3.13"]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v4
        with:
          enable-cache: true

      - name: Set up Python ${{ matrix.python-version }}
        run: uv python install ${{ matrix.python-version }}

      - name: Install dependencies
        run: uv sync --extra dev

      - name: Ruff check
        run: uv run ruff check

      - name: Ruff format check
        run: uv run ruff format --check

      - name: mypy
        run: uv run mypy

      - name: Bandit
        run: uv run bandit -c pyproject.toml -r src

      - name: pytest
        run: uv run pytest

  gitleaks:
    name: Secret scan at tag
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # Stage 2 — build the artifact. Only runs if every verify-ci matrix job passed.
  build:
    name: Build sdist + wheel
    needs: [verify-ci, gitleaks]
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v4
        with:
          enable-cache: true

      - name: Set up Python 3.12
        run: uv python install 3.12

      - name: Build
        run: uv build

      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/

  # Stage 3 — publish the GitHub Release. Only runs if build succeeded.
  publish:
    name: Publish GitHub Release
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Download build artifacts
        uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist/

      - name: Extract release notes from CHANGELOG
        id: notes
        run: |
          # Pull the section for this version out of CHANGELOG.md.
          # Falls back to GitHub's auto-generated notes if missing.
          version="${GITHUB_REF_NAME#v}"
          awk -v v="$version" '
            /^## \[/ { in_section = ($0 ~ "\\["v"\\]"); next }
            in_section { print }
          ' CHANGELOG.md > release-notes.md
          if [ ! -s release-notes.md ]; then
            echo "no-changelog-section=true" >> "$GITHUB_OUTPUT"
          fi

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: ${{ github.ref_name }}
          name: ${{ github.ref_name }}
          body_path: ${{ steps.notes.outputs.no-changelog-section == 'true' && '' || 'release-notes.md' }}
          generate_release_notes: ${{ steps.notes.outputs.no-changelog-section == 'true' }}
          draft: false
          prerelease: false
          fail_on_unmatched_files: true
          files: |
            dist/*
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
